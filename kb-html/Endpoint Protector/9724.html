[UPDATE]- MacOS deployment using Kandji, is not enabling full disk access

If you notice that Full disk access is not granted to Endpoint Protector after the deployment through Kandji please check the privacy settings section where you set the Identifier and code requirements taken from the Guide.

The behavior is caused by adding the identifier in the code requirements, as per some guides you might have followed from Kandji.

Please remove the identifier and match the code from the deployment guide from Endpoint Protector and Full disk access will be set and granted.
