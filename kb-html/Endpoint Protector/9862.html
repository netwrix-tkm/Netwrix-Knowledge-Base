[UPDATE]- How to apply an Offline Patch

Overview
If your EPP server is in a closed environment without internet access, you can upgrade it using an Offline Patch instead of the Live Update functionality. Follow the steps below to perform this upgrade.

Instructions

  1. Create a Snapshot:

    • Before upgrading the EPP server with Offline Patches or Live Update, create a snapshot of the server from your platform (VM, AWS, Azure, or GCP) as a best practice.

  2. Contact Support:

    • Open a support ticket to inform us about your current EPP server version and the version you want to upgrade to.

  3. Receive Offline Patches:

    • We will provide you with the necessary Offline Patch(es) to apply for the upgrade.

  4. Upload Offline Patch:

    • Navigate to Dashboard > Live Update > Offline Patch Uploader.
    • Select Browse, choose the Offline Patch file, and click Upload Patch.

  5. Monitor and Complete Upgrade:

    • Wait on the page until a green confirmation message appears.
    • Track the upgrade progress at Dashboard > Live Update. The progress bar should reach 100%, usually within 5-10 minutes, depending on your database size.
    • After the upgrade completes, refresh the web browser page. Verify the updated EPP server version in the bottom right corner of the EPP server UI.

Notes

  • Delay in Availability: There may be a delay between a new GA release and the availability of an Offline Patch for the new EPP server version.
  • Incremental Updates: Offline Patches are incremental like the Live Update functionality. They must be applied one at a time to correctly upgrade the EPP server.
