Debug Logging

Question:

How do I enable and collect Debug Logs from Endpoint Protector Client?

Answer:

To enable Debug Logging for an endpoint machine please follow the below guide:

  1.  Enable "Debug Logging". To enable "Debug Logging" navigate to "Device Control" > "Computers". Select the computer which the issue is occurring then click on "Manage Settings".
  2. Scroll down to the section "Debug Logging". Click on the drop down menu, next to "Set Log Level" and select "Debug" and then click on "Save".
  3. Update the policies on the affected endpoint machine in order to inherit the new setting by right clicking on the Endpoint Protector notifier and then click on "Update Policies Now".
  4. Reproduce the issue.
  5. Open the Endpoint Protector notifier and Press "Upload Logs" under the tab "Troubleshooting", or from the EPP client UI, while the computer is selected click on "Collect Diagnostics".
  6. The Endpoint Notifier will begin to upload the log file to the Server Console. Wait for the message "Job Complete!" to appear.
  7. Navigate to "Reports and Analysis" > "Logs Report" in the Server Console and download the logs uploaded from endpoint machine, or from the EPP client UI, while the computer is selected click on "Go to Diagnostics Data". 
  8. Please submit the .ZIP archive to the Endpoint Protector Support Team.
