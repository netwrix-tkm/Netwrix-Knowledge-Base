[UPDATE]- How to prevent protect EPP client from unauthorized uninstallation

There are two security measures that can be taken to secure the Endpoint Protector Client on endpoint machines.

The first one would be to set an uninstall password, this would require the user to enter the uninstallation password defined by the EPP system administrator before proceeding to remove the Endpoint Protector client from the machine. This works for all operating systems, Windows, Linux and macOS endpoint machines.

The second security measure is to enable "Tamper Mode".

Starting with Endpoint Protector Server version 5.8.0.0, you benefit from an additional security measure that safeguards the Agent integrity, available from the Device Control > Global Settings page. The Tamper Mode setting prevents the Endpoint Protector Agent from unauthorized termination or alteration. Note that this feature only applies to endpoints with Windows Operating System

Please follow the below steps in order to secure the Endpoint Protector client on endpoint machines.

To password protect the Endpoint Protector client, please follow the below steps.

  1. Navigate to "System Configuration" > "System Security" 
    Enter a password under "Security Password for Uninstall Protection" and then click on "Save".

See below screenshot.

The notification "Changes have been saved!" will appear when the changes are saved successfully together with "You have defined an uninstall password" which indicates that an uninstall password is set.

Update the policies on the endpoint manually or wait for the policies on endpoint machines to be automatically updated based on the time set  for "Policy Refresh Interval".

         2. To enable "Tamper Mode"

Navigate to "Device Control" > "Global Settings" > "Tamper Mode" and toggle the switch to "On"

Scroll down to the bottom of the subsection labeled "Endpoint Protector Client" and then click on "Save"

Update the policies on the endpoint manually or wait for the policies on endpoint machines to be automatically updated based on the time set for "Policy Refresh Interval".
