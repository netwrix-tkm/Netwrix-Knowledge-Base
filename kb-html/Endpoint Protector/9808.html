[UPDATE]- How to set up E-mail Alerts

The e-mail alerts can be used in order to send specific logs to the administrator's e-mail.

In order to set up the E-mail Alerts you will have to set the E-mail server settings, accessing System Configuration > System Settings

Select the E-mail Type > enter the Hostname and the SMTP port > Check the "Require SMTP Authentication" check mark > enter the e-mail address and password > Select the Encryption type based on the port selected

In order to test and make sure that the EPP Server can send the e-mails, you should check the "Send test email to my account" check mark.

Once the settings will be saved, a test e-mail will be sent to the administrator.

By default, the No-reply email address is noreply@endpointprotector.com but you can change it to a custom email address by switching the No-reply email address from Default to Custom

Once the E-mail server settings were set, you can create the Alerts by accessing: Alerts > 

->System Alerts - for alerts regarding the system (ex: Server Disk space, Licenses, Password Expiration)

-> Device Control Alerts - for alerts regarding Device Control (ex: Device Connected/Disconnected, File copy, Uninstall attempt)

-> Content Aware Alerts - for alerts regarding Content Aware Protection (ex: Content Threat Detected, Content Threat Blocked)

-> EasyLock Alerts - for alerts regarding EasyLock (Change User Password, Password Login Exceeded, Password Login Failure)

> Click the Create button > Select the Event > enter the Alert name > Select the Administrator which should receive the alert > Save
