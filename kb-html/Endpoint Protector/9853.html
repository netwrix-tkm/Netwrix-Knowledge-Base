Configuring the Audit Log Backup

Question:

How do I configure the Audit Log Backup?

Answer:

The Audit Log Backup menu(System Maintenance > Audit Log Backup) contains 2 important sections: Manual Audit Log Backup and Scheduled Audit Log Backup.

 

Manual Audit Log Backup:

Gives the possibility to perform an one time backup:

 

Scheduled Audit Log Backup:

Gives the possibility to create a reoccurring Audit Log Backup that has a frequency with which it runs(from a few days to X months). This is similar to the Manual Audit Log Backup, but it will run periodically, at a frequency chosen by you.

 

What you will have to select:

  1. What kind of logs you want to backup(in case you want to have an audit only for Content Aware, you can choose only that):


  2. As you intend to delete the logs and backup them, in order not to fill the server up, you will have to leave the following unchecked:


  3. Select which old logs to backup. This means that all the logs older than X days/months will be removed from Server and transformed into backups:


  4. Select the number of logs in one package(you can leave the default 1 mil).
  5. Select the format.
  6. Choose if you would prefer to delete and backup Shadows too. We encourage to include them as the file shadows occupy more space then the logs and could be the main cause for the server disk space increase.

After the Audit Log Backup has run, the archived logs will be present on the bottom of this menu in the Backup List:
