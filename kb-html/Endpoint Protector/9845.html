Setting up a SIEM Integration

Question:

How do I set up a SIEM Integration?

Answer:

If you are looking to Integrate your SIEM server with Endpoint Protector, please make sure the following requirements are met:

  • The SIEM integration has a Private IP if it is on the same network as the EPP server, or it has a public IP address or DNS if it is on a different network.
  • Please make sure that the SIEM server can receive Syslog-ng type of logs.

 

If the requirements are met, please log into the EPP web console:

  1. In the menu go to Appliance -> SIEM Integration.
  2. On the SIEM Integration window please press the Add New button.
  3. In the New Server window please fill in all the required information and remember that you can use either an IP address, a DNS or FQDN.
  4. If the Disable Server Logging is enabled, the logs are sent directly to the SIEM server without being kept on the EPP server.
  5. The Security Token field is optional and is only required by some specific SIEM servers for a successful connection.
  6. If you do not wish to receive any header and just need the data, you can enable the Exclude Headers option.
  7. Make sure to select all Log Types that are required to be sent to the SIEM server.
  8. After you press the Save button, the integration setup is complete and the server will redirect the logs to your SIEM server.

 

Please note that previous logs will not be sent to the SIEM server, only the ones received after the integration was set up.
