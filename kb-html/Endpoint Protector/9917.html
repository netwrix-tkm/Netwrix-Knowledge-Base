    <h2>Overview:</h2>
    <p>Executing a backup restore for Endpoint Protector is essential for ensuring data security and system integrity, especially when using the eDiscovery module.</p>

    <h2>Question:</h2>
    <p>What are the necessary steps and considerations for effectively performing a backup restore for Endpoint Protector?</p>

    <h2>Resolution:</h2>

    <h3>Backup Process:</h3>
    <ul>
        <li>Navigate to <strong>System Maintenance -> System Backup v2 -> System Backup Information</strong></li>
        <li>Create a System Backup v2:
            <ul>
                <li>Press Create to access System Backup Information. Provide a name and a brief description for the backup you are creating.</li>
            </ul>
        </li>
        <li>Select <strong>Save</strong>: Click 'Save' to initiate the backup process.</li>
        <li>System Backup Key Storage: A pop-up window will display a System Backup Key, which is essential for importing and restoring the backup. Ensure you store this key securely (e.g., N07K1IAC0W00C4S4) for future restoration needs.</li>
        <li>Confirm Key Storage: Select 'Yes, I saved the System Backup Key' to confirm youve securely stored the key.</li>
        <li>Managing the Backup:
            <ul>
                <li>The system backup, excluding logs, will be created and accessible in the 'List of System Backups'.</li>
                <li>This section details the Backup Name, Description, Server Version, File Name, File Hash, File Size, Creation Date, and Status.</li>
                <li>Under 'Actions', you can view, download, or delete the backup as needed.</li>
            </ul>
        </li>
        <li>Preparation for Restore:
            <ul>
                <li>Its advisable to utilize the Audit Log Backup prior to the restore since logs wont be retained. Also, download any previous system backups as only the system backup you are about to import and restore will remain.</li>
            </ul>
        </li>
    </ul>

    <h3>Restore Process:</h3>
    <ul>
        <li>Preparation for Restore:
            <ul>
                <li>Ensure the Audit Log Backup is utilized as logs wont be retained. Download previous system backups since only the system backup you import and restore will remain.</li>
            </ul>
        </li>
        <li>Navigate to <strong>System Maintenance -> System Backup v2 -> System Backup Information</strong></li>
        <li>Select <strong>Import and Restore (Migrate)</strong>:
            <ul>
                <li>From Import: Choose File..., select the System Backup v2 saved during the backup process.</li>
            </ul>
        </li>
        <li>Enter the System Backup Key into the provided box.</li>
        <li>Select <strong>Import</strong>.</li>
        <li>Monitor the Status: In the List of System Backups, you will see the status as "Generating" in red letters for the backup in question.</li>
        <li>Re-login: The Endpoint Protector console will log you off. Log back into the Endpoint Protector Console.</li>
        <li>Restore Logs: Use the Audit Log Backup feature if necessary to restore logs.</li>
        <li>Additional Step for eDiscovery Module Users:
            <ul>
                <li>For those using the eDiscovery module, employ the re-read rights and settings for all computers feature after the backup restoration (Dashboard &gt; System Status -&gt; System Functionality).</li>
                <li>This final step ensures accurate correlation between previously discovered items and actions (such as Encrypt, Decrypt, Delete, etc.) that can be performed on these files.</li>
            </ul>
        </li>
    </ul>

    <p>By following these steps, you will achieve a successful backup and restoration process for Endpoint Protector, ensuring data accuracy and integrity throughout.</p>
