How to Deny/Block Applications With Content Aware Protection

To Block/Deny an application from running on an endpoint please follow the steps listed below:

  1. Identify the process name first, in Windows the process name will be listed under the "Details" tab of Task Manager and under the "Process  Name" column in "Activity Monitor" for macOS clients. Windows > Task Manager macOS > Activity Monitor Once the  process name been identified, navigate to "Denylists and Allowlists" > " Denylists" > "Applications" then click on "Add". 
  2. Proceed to enter the name, description. In the box titled "Application & CLI Command" enter the process name exactly as it appears on  the endpoint Operating System. Then click on "Add to Content".
  3. The process name will appear unchecked in the right hand box titled "List of Application & CLI Command". Check the  newly added process name and then click on "Generate".
  4. The application list will now be added and ready to be selected within the Content Aware Policy.
  5. Create a Content Aware Policy or use an existing one, navigate to "Policy Denylists" > "Applications" and select  the newly created application list. Select the endpoints which the policy needs to apply to under "Policy Entities", save the Content Aware Policy and then wait for the policies to be updated on the endpoints.
  6. Endpoint Protector through Content Aware Protection will now prevent the application from running when the user attempts to start the application again.
