Setting up an "Offline Temporary Password"

In environments where the "User Remediation" feature cannot be used, the "Offline Temporary Passwords" feature provides an alternative way to bypass a policy.

Within the Endpoint Protector Management Console, administrators can generate Offline Temporary Passwords (OTPs) to grant users temporary access rights. This feature is useful in situations where only temporary access is required or when there is no network connection between the protected computers and the Endpoint Protector Server (for example, when User Remediation is unavailable).


You can generate an Offline Temporary Password for the following entities:

 

- Device (a specific device)

- Computer and User (all devices)

- Computer and User (all file transfers)


Each password is linked to a specific time period and is unique to a particular device and computer. The same password cannot be used for a different device or computer, nor can it be reused (except in the case of Universal Offline Temporary Passwords). Available time intervals include: 15 minutes, 30 minutes, 1 hour, 2 hours, 4 hours, 8 hours, 1 day, 2 days, 5 days, 14 days, 30 days, or a custom duration.
