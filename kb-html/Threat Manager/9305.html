Netwrix Threat Manager (NTM) not receiving events.

Symptom:

Threat Manager does not appear to be gathering threat events. 

Cause:

The App Token within NRM is no longer valid or working between the needed Application servers. 

Resolution:

With the prior App Token no longer working a new Token will need to be created. 
Netwrix Threat Manager v2.8 > Administration > Configuration Menu > Integrations Interface > App Tokens Page> Generate an App Token
 


This new App Token can now be updated in the sending applications servers.

  • NAM Netwrix Activity Monitor v7.1 > Administration > Output Types > Threat Manager Tab  
  • NAM File system is UDP no Token needed Syslog output
  • NTP Netwrix ThreatPrevention v7.4 > Administration > Netwrix Threat Manager Configuration Window > Event Sink Tab

IMPORTANT: Being that Netwrix Threat Manager (NTM) formally StealthDEFEND (SD) receives data from the following applications, they need to be checked proper functionality.

  •  Netwrix Activity Monitor (NAM) formally Netwrix Activity Monitor (SAM) Output for Monitored Hosts
  •  Netwrix Threat Prevention (NTP) formally StealthINTERCEPT Server (SI) Agents.
