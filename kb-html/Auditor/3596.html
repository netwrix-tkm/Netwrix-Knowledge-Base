How to exclude files, folders or shares from auditing by Netwrix Auditor for File Servers?
If you do not want to audit every file and folder on the file server's shares or if you are monitoring the whole server but want to omit a specific share, you can achieve that by using a special omit file.
  1. Navigate to the product installation folder via Windows Explorer (by default: C:\Program Files (x86)\Netwrix Auditor\File Server Auditing)
  2. Open omitcollectlist.txt file in any text editor
  3. Add the paths you do not want to be audited. For example:
    • Exclude a specific file in a share: *,,\\\\server\\share\\file
    • Exclude all files and subfolders in a share: *,,\\\\server\\share\\*
  4. Save changes

Note: The above changes will not take effect until the next data collection. Also, this would only prevent excluded data from showing in reports, but will not suppress warnings/errors that happen during data collection. For instance, "Access denied" errors will not disappear.
