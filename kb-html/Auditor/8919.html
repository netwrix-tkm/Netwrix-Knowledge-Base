LCM_Snapshots Folder Occupies Too Much Disk Space

Question

The LCM_Snapshots folder is growing too large in size (in NA v10 and newer). Why is this happening?

Answer

If Working Folder was not excluded from the monitoring scope of the antivirus suite, the LCM_Snapshots folder might grow large in size. Review the AV exclusions in the environment ? refer to the following article: Antivirus Exclusions for Netwrix Auditor.

NOTE: The LCM_Snapshots folder is a duplicate of the FSA collector snapshot and is intended exclusively for calculating the license metrics for each source (items+omits). Metrics are recalculated once every 7 days ? the file deletion should occur automatically 20 minutes after each source is calculated ? a copy of the snapshot is unloaded from memory.

In case no AV exclusions are introduced, and the antivirus suite blocks the LCM_Snapshots folder files, the antivirus suite also blocks the working snapshot files for AR and SIT.
