Include Change Password Events by Users via Ctrl+Alt+Del Combination in Netwrix Auditor Reports

Question

Is it possible to include change password events performed by users through the ctrl+alt+del key combination into product reports?

Answer

Yes, it is possible. Such events are omitted by default. To include them in product reports, follow the steps below:

  1. Navigate to the Auditor for Active Directory installation folder: C:\Program Files (x86)\Netwrix Auditor\Active Directory Auditing by default.

  2. Locate the omitproplist.txt file and open it with a text editor.

  3. Locate the *.PasswordChanged line and add a "#" sign at the beginning so it looks like #*.PasswordChanged.

  4. Save your edits.

Once these changes have been applied, password resets performed by users will be shown in the product change reports.
