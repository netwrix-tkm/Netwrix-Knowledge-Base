Error: Cannot Insert Duplicate Key in Object

! This article is currently under review. Some of the information below may not be up to date.

Symptom

The following warning appears in the Netwrix Auditor System Health log:

Upload failed due to the following error:
Violation of PRIMARY KEY constraint 'item name'. Cannot insert duplicate key in object 'object name'. The duplicate key value is (1, 1).
The statement has been terminated.

Cause

This message occurs if the SqlCalculateRequiredAttributesCommandTimeout was set incorrectly in the SPODDCCoreSettings.xml file.

Resolution

To resolve the issue, you need to set the timeout to 7200. For that:

  1. On the computer that hosts Netwrix Auditor Server, stop the Netwrix Auditor NDC Provider service.

  2. Navigate to C:\ProgramData\Netwrix Auditor\Data Discovery and Classification\ and open the SPODDCCoreSettings.xml configuration file with any text editor, e.g., Notepad.

  3. Locate the SqlCalculateRequiredAttributesCommandTimeout the line and set the timeout to 7200.

  4. Save the file.

  5. Start the Netwrix Auditor NDC Provider service.
