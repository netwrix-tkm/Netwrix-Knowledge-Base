Troubleshooting File System Action Module  KeepSessionFiles xml option

Symptoms and cause

After executing a File System Action you may encounter errors like "Failed to import the results into a database" or other similar errors. These errors often indicate a missing column or invalid data type in the source table for the action.
When any File System Action is executed, an input1.db and output1.db file are created for each session, but by default these are deleted once the execution completes. These files are useful for troubleshooting and should be retained and provided to R&D when escalating issues with File System Actions.

Resolution

Follow the steps below to allow these files to be retained:
  1. Right click on the job and select Explore Folder
  2. In the job folder, open the ActionTasks.xml file with your preferred text editor
  3. Locate the <KeepSessionFiles> option for the specific action task and set this option to true (case sensitive).
  1. Save the ActionTasks.xml file then run the action.
  2. Once the action completes navigate to %SAINSTALLDIR%Actions\FileSystem and open the folder that has the timestamp of when the action was executed as its name
 
  1. You should see two files within this folder:
    • [HOST].input1.db.zip
    • [HOST].output1.db.zip
  1. Copy these two files and provide them to R&D when escalating
