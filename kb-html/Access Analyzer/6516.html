Access Violations

Summary: Access Violation when running Data Collectors that have an .exe associated to them

Issue: Experiencing access violations

Instructions: There are a few data collectors that have associated .exe files with them:
Blackberry Data Collector, 
Powershell Data Collector, 
ExchangePS DC, 
Exchange2k DC, 
Smartlog DC

This issue can be caused by having UAC turned on, or the account running Access Analyzer not having sufficient local administrator privileges.  

A work around for this is to set all exe files in the Private Assemblies folder to always Run as Administrator. 
go to %sainstalldir%\Private Assemblies
search for .exe
for each one

right click > properties
> compatibility tab
> change settings for all users button
> check the "Run this program as an administrator" box 
> OK > OK


You can do this for the StealthAUDIT.exe, as well.

Module: Access Analyzer - DC - Blackberry;Access Analyzer - DC - Exchange2k;Access Analyzer - DC - ExchangePS;Access Analyzer - DC - PowerShell;Access Analyzer - DC - Smartlog EventLog;Access Analyzer - DC - Smartlog IIS
