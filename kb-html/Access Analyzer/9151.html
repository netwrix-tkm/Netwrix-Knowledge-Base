Custom Attribute Warnings in Active Directory Inventory Scan Job

Symptom

The Active Directory (AD) inventory scan job prompts the following warning in Netwrix Enterprise Auditor:

The custom attribute %attribute_name% was requested, but not set in domain %domain_name% for the %class_name% class.
A full ADI scan may be required.

NOTE: The warnings include AD schema attributes and classes. These include but are not limited to AdminCount, msDS-AllowedToActOnBehalfOfOtherIdentity, msDS-AllowedToDelegateT, or PrimaryGroupID attributes and Computer object, User object, or Group object classes.

Cause

The AD inventory scan job searches for common AD schema attributes by default. The job prompts corresponding warnings if your environment does not implement these attributes.

Resolution

This behavior is expected if your environment does not implement these attributes. You can ignore the warnings or follow the steps below to remove custom attributes used in AD scans:

  1. In Jobs, navigate to .Active Directory Inventory > 1-AD_Scan > Configure.

  2. Right-click Queries and select Properties.

  3. In the Data Source tab, click Configure.

  4. In the Custom Attributes step of Active Directory Inventory wizard, select the affected attribute and click Remove.

  5. Click Next > Finish > OK.
