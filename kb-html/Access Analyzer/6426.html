NetApp 7-mode activity monitoring is not working when the SAM agent is installed on Windows 2019 machine

 

Summary: NetApp 7-mode activity monitoring is not working when the SAM agent is installed on Windows 2019 machine

Issue: You are attempting to monitor NetApp in 7-mode via a Stealthbits Activity Monitor agent running on a Windows 2019 machine. When trying to use the connect button during configuration setup, you see messages about a being unable to open a TCP session or certificate error. [Note that other issues can also cause the certificate error. Try with HTTP ignore certificate errors option.]  Communication from the agent to the NetApp cannot be established, so scanning is not possible.

Instructions:  NetApp 7-mode by default requires SMB1 protocol. Windows 2019 by default disables SMB1 protocol. The preferred method to resolve this issue is to use the setting options cifs.smb2.client.enable on in the NetApp configuration to enable SMB2 communication. This method is the most secure.  (You can also enable SMB1 on Windows 2019 and restart the SAM agent. This is less secure and not the preferred option.)

Product: SAM & NetApp
