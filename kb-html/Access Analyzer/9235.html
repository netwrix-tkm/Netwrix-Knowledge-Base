FSAA Proxy Scans in Non-trusted Domains

Overview

In Netwrix Enterprise Auditor v11.6, the communication between the FSAA Data Collector and the FSAA Applet is secured via the HTTPS protocol. This article covers the steps to correctly set up the FSAA Proxy service in your environment to scan the non-trusted domain resources.

Instructions

IMPORTANT: Allocate a separate service account with local Administrator rights to implement the FSAA Proxy scans in your non-trusted domain.

Refer to the following steps:

  1. Manually create and set up certificates for the FSAA Proxy using the allocated service account. Refer to the following article for additional information: FSAA Applet Certificate Management Overview  FSAA Manual Certificate Configuration · v11.6.

  2. In the target server, run Services > select the StealthAUDITFSAA service > switch the Log On account to the service account.

  3. Add a connection profile with the corresponding credentials of the service account to use for the FSAA Proxy scans. Refer to the following article for additional information: Connection  Create a Connection Profile · v11.6.

  4. Run the FSAA Proxy scan job.
