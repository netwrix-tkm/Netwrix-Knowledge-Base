Logs Not Clearing Automatically 혰 Insufficient Space in Access Analyzer Console Drive

Symptoms

  • Insufficient space in the Access Analyzer console drive.

  • Access Analyzer FSAA logs consume a large portion of disk space. For additional information on Access Analyzer logs, refer to the following article: Access Analyzer Logs.

Cause

FSAA logs are not clearing up automatically.

Resolution

  1. Download the Log Maintenance job group attached to the article, and add it to Access Analyzer.

    NOTE: It is recommended to add the job group to the Sandbox folder, if available.

  2. Set the target host list for the job group to match the location of the FSAA logs to be cleared (e.g., either localhost or the FSAA proxies).

  3. In the 1- LogCleanup job, configure the query to the FSAA location. If Access Analyzer or proxies are using different paths, job query will need to be run separately against Access Analyzer and proxies, modifying the query accordingly depending on host target.

  4. Optional. In the 1-LogCleanup job homepage, configure the How many days of logs to keep from being removed setting. The default settings is 7 days.
