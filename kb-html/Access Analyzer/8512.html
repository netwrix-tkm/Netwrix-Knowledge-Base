Hotfix: SAM_6.0_029

Summary:
1. NetApp monitoring changes:
               - Better handling of bursts of activity when events come in batches.
               - Faster processing, better utilization of CPU cores.
               - Request Abort timeout in the External-Engine is set to 10 seconds instead of default 40.
               The new version is at least 2 times faster in the worst-case scenario (a large batch of activity from a single user). On average, we expect it to be 7 times faster.
               2. Improved performance of syslog outputs
               3. Linux: Deployment of the agent may hang during SSH upload
               4. Active Directory: Improved noise suppression when only 'lastLogon' attribute is changed
Behavior Before Fix:
1. The product could not effectively handle bursts of activity events from NetApp. The product did not utilize available CPU cores effectively.
               2. Event reporting to syslog could be slow.
               3. Installation of the agent could hang while copying the installation package via SSH.
               4. When 'logonCount' reaches the maximum value of 65535, it is not updated anymore. The noise filter suppresses events only when both 'lastLogon' and 'logonCount' are modified.
Behavior After Fix:
               1. The product process events from NetApp significantly faster, using CPU cores more effectively. It is able to consume large batches of events quickly.
               2. The syslog processing pipeline was optimized.
               3. The installation package is copied to the home directory, respecting the quote mode.
               4. Changes to 'lastLogon' only (when 'logonCount' reaches the maximum) are considered noise activity.
