Access Analyzer process running scheduled tasks did not end

Summary: Scheduled task appears to be stuck, how to verify and resolve

Issue: While in operating system's Task Scheduler, notice a scheduled task is running longer than it usually does. Job may be stuck, to verify and/or resolve, follow the steps below.

Instructions:

To verify that the job is actually stuck, check the job statistics.  If the job's last run is the same time as the scheduled task's start time, and if it has a finish time, it is possible that the process is stuck.

To remediate the issue

1.    Right click on the scheduled task in task scheduler and select end.

2.    Check the task scheduler for the corresponding Access Analyzer process.  If you add the command line column to the processes tab, it will show you the job that is running.  End the process of the job causing the issue.

3. Inspect the folder of the job and remove any running.lck files that may be present.  Right click on the job and select explore folder.  If a running.lck file is present, delete it.
