activitymonitor-7.1.258

 

Summary:

  1. NAM - User SID resolves incorrectly
  2. SUVP 25-02 support

Behavior Before Fix:

  1. Previously the search behavior was to resolve the account SID that started with the SID value. And this resolved incorrectly because there was another account starting with the same values.
  2.

      • Couldn't resolve LDAP_CONN::BindRequest
      • Couldn't resolve KdcGetTicket

      • Couldn't resolve I_GetASTicket

    • NTLM and AD Replication events will be lost or incomplete
