Merge Statement Conflicted with Foreign Key Constraint Error in SPSEEK Bulk Import Job

Symptom

When attempting to run the SPSEEK Bulk Import job, the following error is prompted:

[C:102] Unable to perform bulk import
Error: The MERGE statement conflicted with the FOREIGN KEY SAME TABLE constraint %FK_constraint%.
The conflict occurred in database %database_name%, table %table_name%.
The statement has been terminated.

Cause

The foreign key constraint conflict was caused during the latest scan.

Resolution

Drop the SharePoint tables and run the scan again ? to drop the tables, run the SP_DropTables job ? refer to the following article for additional information: SharePointAccess Data Collector ? SPAA Drop Tables & Views Workflow ? v11.6.
