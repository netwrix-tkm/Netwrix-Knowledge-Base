How To Enforce group naming convention and input validation with regular expressions

Applies To:

GroupID 11

Summary:

Naming standards play a bigger role in your IT enterprise than you may think. It becomes even more important when users are given access to Active Directory via the User portal.

GroupID administrators can enforce a naming convention for groups that are created using the User portal by defining regular expressions. This involves two main steps:

  • Create a display type
  • Associate this display type with the desired field

Create a display type

  1. In the GroupID Admin Panel, select Applications, and against your desired portal/application, click on the 3 dots button and select Settings.

  1. On the next page, under Design Settings, select your Identity Stores name to expand it.

  1. Select Custom Display Types and click the Add button.

  1. On the New Display Type page, enter a name for the display type in the Name box. This name should reflect the purpose of the display type, for example, Regular Expression Policy 1.
  2. From the Type list, select the Textbox option and click OK. Fill out the other options based on information below.

Default Value

Specify a default value that will be displayed in the textbox.

Regular Expression

Specify a regular expression to perform a validation on the data entered in the textbox.

Error Message

Type the text that would be displayed as an error message when a portal user enters data in the text box that does not conform to the regular expression.

 

  1. Click OK and then click the Save button to save your changes.
    This new display type will be listed under the Custom Display Types area, along with the already defined display types.

Associate the display type with a field

The next step is to associate the display type with the required field. Let's suppose you want to associate the display type with the Group Name field on the Create Group wizard in the User portal.

  1. In the GroupID Admin Panel, select Applications, and against your portal/application where you just added the new display type, click on the 3 dots button and select Settings.
  2. On the next page, under Design Settings, select your Identity Stores name to expand it.
  3.  Scroll down and click on the Create Object option. Under the Select Directory Object option, select Group from the drop-down list.
  4. In the Name list, select General and click Edit.

  1. On the Edit Design Category page, select Group name in the Fields area and click Edit.

  1. On the Edit Field page, select the display type you created (namely, Regular Expression Policy 1) from the Display Type drop-down list.

  1. Click OK and then click OK for 2nd time to close the dialog boxes, and then Save the changes by scrolling down the page.
    Now, this display type is linked to the Group name field on the Create Group wizard in the portal.

Final Result:

The naming convention we defined using a regular expression was to ensure that group names only contain alphabetic characters (lowercase and uppercase), without spaces and special characters. So, when a user tries to enter a group name with spaces or special characters, the portal will not validate the name and will display the standard example that we specified. It is as follows:

Some examples of regular expressions are discussed below.

Example 1:

Ensure that the desired field only contains alphabetic characters (lowercase & uppercase) and numeric digits (0-9). In addition, the length must be constrained between 1 to 25 characters.

Regular Expression:
^[A-Za-z0-9-]{1,25}$

Example 2:

Ensure that the desired field only contains alphabetic characters (lowercase & uppercase), numeric digits (0-9), underscores (_), hyphens (-), asterisks (*), commas (,), and space characters. In addition, the length is constrained between 1 to 30 characters.

Regular Expression:
^[A-Za-z0-9-,_*\s]{1,30}$

Example 3:

A phone number in North America starts with three numbers, which represent the area code, followed by seven digits split into sections of three and four (which represent the local number), so the number looks like (555) 123-4567. We have been facing this problem where users do not follow the pattern or add an extra number at the end. We want to make sure that users follow the standard pattern while entering phone numbers.

Regular Expression:
^\ (\d\d\d\) \d\d\d-\d\d\d\d$

Example 4:

The following regular expression will accept telephone numbers in the format: 1 555 555 5555.

^[01]?[- .]?(\([2-9]\d{2}\)|[2-9]\d{2})[- .]?\d{3}[- .]?\d{4}$
