How To: Ensure that groups have a unique display name

Applies To:

Directory Manager 11 

Business Case:

We have a requirement that each group that our users create from the Application portal should have a unique display name and it should not be the same as that of an existing group.

Methodology:

We need to make some changes in the design of the application portal to require users to provide a unique display name for each group.

Steps:

  1. In Directory Manager Admin Center, navigate to Applications -> Your Portal -> Settings.

  1. Next, click on the identity store name under the Design settings section.

  1. On the Properties tab, select Group as the directory object, select General, and click Edit.

  1. On the Edit Design Category box, select Display name and click Edit.

  1. On the Edit Field dialog box, displayName is selected in the Field box.
    For Display Type, Text is pre-selected. Select UniqueText here.

  1. Click the Ok button and then Save on the outer window.
  2. If you want to apply this to Smart Groups as well, select Smart Group as the directory object in step 2 and apply the same steps to it as well.

Now, when a user tries to create a group from the Self-Service portal with an existing displayname, the portal will not allow it.

If you have multiple portals, apply the above-given steps to all your portals.
