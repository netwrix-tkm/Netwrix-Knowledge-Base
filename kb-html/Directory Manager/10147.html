Info: How GroupID classifies groups and types

Applies To:

GroupID 11

Summary:

GroupID classifies groups into two broad categories: Unmanaged and Managed.

More Information:

Unmanaged Groups

An unmanaged group, also called a static group, is one you would normally create in a directory (for example, by using the Users and Computers option in Active Directory). Though such groups can be created using GroupID application portal, GroupID does not support dynamic updates to them. Any changes to the membership have to be done manually.

Managed Groups

A managed group, also known as a Smart Group, is one that dynamically maintains its membership based on rules. These rules are applied in the form of a user-defined LDAP query. This query is defined once and scheduled for membership update using a Smart Group Update job. When the Smart Group Update job runs, it applies the defined rules to update the group's memberships.

A dynasty is also a collection of Smart groups that are created based on the values of the defined attribute of the dynasty. They are governed by a parent group called as 'Parent' dynasty. For the updates of these groups, you only have to update the parent dynasty. When you update the Parent dynasty, its middle and leaf dynasties are automatically updated. 

This automated group management allows administrators to easily maintain large distribution lists and security groups without having to manually add or remove members.

You can create and manage Smart Groups using the GroupID application portal.

Security Types of Groups:-

For both Unmanaged and Managed groups, GroupID classifies the groups into 3 security types. These security types define the access level for a group. These 3 types are as follows:-

 Private groups:

A private group is one whose membership is determined solely by the group owner. The owner determines who can join and leave the group. Additional owners can also manage group membership. However, users cannot even request to join or leave a private group. By default, the groups that are created directly in your directory, i.e., unmanaged groups, are classified as Private. You can change their security type from GroupID.

 Semi-private groups:

Semi-private groups, like private groups, have an owner who determines membership, except that members can request to join and leave the group.

Public groups:

A public group is open to all users. Users can join and leave a public group without requiring owner permission. The administrator can configure email notifications to be sent to the group owner when a member joins or leaves the group.
