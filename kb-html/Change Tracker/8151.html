How to stop Change Tracker recreating deleted default groups?

Have you created custom groups for your agents? Have you deleted some or all of the default OS groups which come with the software as default? Do the groups keep re-appearing on a regular basis? - GREAT! Then you'll need to follow the below guide.

By default the Change Tracker console will re-create default groups and their attached configuration templates and compliance reports whenever the Netwrix application pool is recycled through IIS, which by default is every 29 hours. In order to stop them from happening, please do the following:


 - Log into Change Tracker

 - Browse to Settings

 - Browse to System Settings

 - Select Show Advanced Options

 - Select Add Config Item

         Enter Description: CreateDefaultGroups

         Enter Value: No

 - Select Update


Once this has been set, this should stop the default groups from being recreated.
