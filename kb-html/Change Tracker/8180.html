How to Stop Change Tracker from Blocking IP Address

This problem occurs when a user fails a login 3 times or more from a remote device. Change Tracker will block the IP address of that device for some time.


Resolution

This problem can be resolved, and can be modified to instead lock out the account that the user is trying to log into after 3 unsuccessful logon attempts instead. 


Step-by-Step

  1. Open Command Prompt (Admin Privilages)
  2. Type iisreset /stop (This will stop IIS). Do not close Command Prompt, for now, please minimize it.
  3. Open Notepad (Admin Privilages)
  4. Click File > Open > Navigate to C:\inetpub\wwwroot\Change Tracker Generation 7 Hub
  5. Select "appSettings.config" and press the "Open" button

  6. Upon opening the file you should see something similar to below:











  7. In the configuration, notice the "  " Value. This value is responsible for the behavior of failed authentications. Below are the explanations of what changing this value will result in:

             = This will block the IP after 3 failed attempts for unspecified amount of time (This is the default setting)
              = This will lock the account out for 10 minutes after 3 failed attempts

  8. Replace value="false" with value="true" if you prefer that Change Tracker lock the account out for 10 minutes after 3 failed attempts rather than blocking the IP address.
  9. Once done, click on File > Save and close out the document. Re-open to make sure the settings are retained.
  10. Back to the command prompt, Type iisreset /start (This will start IIS).


Thats it! your Change Tracker server should no longer be blocking IP Addresses.
