How to Export a Collection from MongoDB

This article is for when you need to export/backup specific collections from MongoDB. In some cases. Exporting a collection may be useful to send over to Development when you've investigated a problem lies within MongoDB.

Step-by-step guide

  1. First access the Hub machine and determine where MongoDB is installed. Default location is:C:\Program Files\NNT Change Tracker Suite\Gen7\MongoDB
  2. To view collections, Open Command Prompt as Admininistrator and Change Directory to the "bin" folder in MongoDB
  3. cd C:\Program Files\NNT Change Tracker Suite\Gen7\MongoDB\bin (If needed to change drives, enter the commandX:, replace X with your Drive Letter)
  4. Next enter the following command to enter Mongo:
  5. .\mongo.exe
  6. Enter the following command: show dbs
  7. Next, enter: use NNTHubService
  8. Enter command: show collections


  9. Once this is completed, you will be able to see all collections located in MongoDB.
  10. Enter the following: exit
  11. To export a collection, enter the following command (Edit according to what is needed)
  12. mongodump -d NNTHubService -c ENTER-COLLECTION-NAME -o "C:\ENTER PATH HERE"
  13. In the "ENTER-COLLECTION-NAME" field, enter the collection you wish to export.
  14. In the "C:\ENTER-PATH-HERE" field, enter path you wish to export to.


The following below is a list of All collections in MongoDB:

AgentTaskQueue, AgentTasks, BackgroundTaskQueue, BackgroundTasks, Counters, DefaultFilters, EventStats, Events, Hashes, HubBackups, HubConfigData,

IncomingEventQueue, IpAddressActivity, Licensing, NotificationQueue, Organizations, PendingNotifications, PerformanceData, PlannedChangeDefinitons, PlannedChangeInstances,

Policy, RemoteCredentials, SavedUserData, Scheduled Policy, Sequences, SystemDirectory, UserAuth, UserAuthDetails, UserRoles, fs.chunks, fs.files
