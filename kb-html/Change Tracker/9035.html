What Are Log Rotation Parameters for HubService, SyncConnector and Windows Agents in Change Tracker?

Question

What is the log rotation values for HubService, SyncConnector and Windows Agents in Change Tracker and where to configure them?

Answer

The HubService-log file begins when service is started and stops at when exceeds the maximumFileSiz value specified in C:\inetpub\wwwroot\Change Tracker Generation 7 (NetCore) Hub\log4net.config file.

Specifically in the following rows:

maxSizeRollBackups value="1"
maximumFileSize value="10MB"

The maxSizeRollBackups parameter indicates how many hubservice-logs will be stored. After hitting the maximumFileSize limit, it creates a copy with a copy number at the end.

The maximumFileSize is how big a hubservice-log can grow to before a new hubservice log will be created.

Default configuration here is 1 backup copy per 10MB.

For SyncConnector, the configuration file is located here: C:\Program Files\NNT Change Tracker Suite\Gen7SyncService\Gen7SyncService.exe.Config.

It shares the same parameters as the HubService config file.

For Windows Agents, the default location of the configuration file is: C:\Program Files\NNT Change Tracker Suite\Gen7Agent (NetCore)\log4net.config. It also shares the same parameters.
