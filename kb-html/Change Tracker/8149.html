How to Configure Recursive File Content Monitoring?

You can monitor for file-content changes alongside FIM using our File Integrity Monitoring tracker but you will need to make some changes to a Change Tracker config file and the configuration template attached to your devices.


By default the FIM/File-content combination is not enabled on the web console. In order to make this available you will need to do the following:


  • Connect to the server hosting your Change Tracker server via RDP
  • Browse to the following folder: C:\inetpub\wwwroot\Change Tracker Generation 7 Hub\bin\Filters
  • Edit the following file: default.xml
  • Find the following chunk of text and remove the text highlighted in red


 

  !--filetrackerchangesfilter name="filecontentschangesonly" description="File attributes (including contents) only"priority="low" source="globalLow" watchdirectories="false" watchfiles="true" created="false" length="true" lastaccess="false" lastwrite="true" filename="true" directoryname="true" attributes="true" security="true"  datastreams="true" content="true" />-->


  • Save and close the file
  • Log off from the RDP session


Now this has been done, you will need to apply this FIM/File-content filter to your devices configuration template. In order to do this you will need to do the following:


  • Log into Change Tracker
  • Browse to Settings
  • Browse to Configuration Templates
  • Select the appropriate configuration template from the list and select the Edit button posted next to it
  • Browse to the FIM File Integrity tracker
  • Select the Tracked Folder entry that you wish to monitor for file content changes recursively and select the Edit button posted next to it
  • Under the Tracked Attributes column, select the drop-down menu and select the File attributes (including contents) only option
  • Press Update
  • Press Save Changes at the top of the screen

Once the changes made to the template have been saved, an updated version of the config will be sent to the agent and a new baseline scan will run. Please allow some time for this scan to complete before making any test changes to files within the directory as they may be captured as part of the baseline and no reported as a change.
