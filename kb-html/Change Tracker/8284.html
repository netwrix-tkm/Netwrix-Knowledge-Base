How To Turn Off Bulk Email Notifications (Force Change Tracker to send one email for each change instead of compiling into one email with multiple changes)

The default behavior of Change Tracker when sending real time email notifications is to wait and compile multiple changes into a single email. In some instances, it may be necessary to have each change separated into its own email address. Below is a guide on how to force Change Tracker to send each change as its own email.

Step-by-step guide

Please follow the below carefully:

  1. Stop IIS ( You can do this by opening an administrative command prompt and running the following command: iisreset /stop )
  2. Navigate toC:\inetpub\wwwroot\Change Tracker Generation 7 Hub\bin\Configs
  3. If your localhost.json file is in a default configuration and you have not split your MongoDB to a different server, then you can replace your localhost file with the preconfigured one here:localhost.json
  4. Start IIS ( You can do this by opening an administrative command prompt and running the following command: iisreset /start)


If you prefer to edit your existing localhost file instead of replacing it with the one above, please insert the following lines of code at the end of the file (in instances where your database has been split off to a different system, the below method will be required):

 
,
  "pipeline" : {
	"enableBulkNotifications" : "no"
	}
