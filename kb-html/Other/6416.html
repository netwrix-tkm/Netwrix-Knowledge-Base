Hotfix: SA_10.0_030

Summary:  Duplicate object names in a SQL Source would cause errors and interruptions when importing into the AIC.

  • AIC imports now more properly handle situations where multiple tables in different schemas have the same name
  • AIC imports now more properly handle situations where deleted objects have the same name as existing objects.

Before Fix:  The two situations detailed in the summary could cause the error: The MERGE statement attempted to UPDATE or DELETE the same row more than once.  This happens when a target row matches more than one source row.

After fix:  AIC imports now ignore deleted objects and use schemas for table names to avoid conflicts.
