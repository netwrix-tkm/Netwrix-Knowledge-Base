How To: Set Do Not Notify feature for Additional Owners in GroupID.

Applies To:

GroupID 11  GroupID Portal, Management Shell

Business Scenario:

We have multiple additional owners set for different groups. Currently, all the additional owners receive notifications of changes being made to the group. However, some of these additional owners do not wish to receive notification emails. Is there a way to restrict some additional owners from receiving the email notifications?

Solution:

Restricting individual additional owners from receiving email notifications is possible in GroupID. We can achieve this using the Do Not Notify feature available within GroupID. In the current GroupID architecture, the Do Not Notify feature is a part of Additional Owner attribute and its value is dependent on it.

Furthermore, when Do Not Notify is set for an additional owner, it also includes the workflow-triggered actions. This would mean that if someone requests to join the group, the additional owner who has set Do Not Notify will not receive the notification emails.

There are three different ways through which we can set Do Not Notify setting in GroupID.

  1. Using GroupID Portal
  2. Using GroupID Management Shell

Steps:

Using Self-Service Portal:

  1. Log into the GroupID Portal.
  2. Go to the Properties of the Group for which the Do Not Notify needs to be set.
  3. Click Owner tab. There will be a checkbox available next to the Additional Owners in the grid.
  4. Check this box to set Do Not Notify for the required additional owners and click on Save.

Using GroupID Management Shell:

The Do Not Notify feature can also be set using GroupID Management Shell using the NotifyOptOutAdditionalOwners parameter. Since the Do Not Notify feature is a part of the Additional Owner attribute, the NotifyOptOutAdditionalOwners parameter cannot be set independently.

In the Management Shell, it can only be set while modifying the Additional Owner attribute. This means that we can either set Do Not Notify while adding new Additional Owners or by modifying the value for already available Additional Owners.

Note:

Please note that before running these cmdlets in your production environment, it is recommended to first run them on some test groups and verify that the results are as per your requirement.

 

 

Steps:

  1. Launch the GroupID Management Shell.
  2. To set Do Not Notify for new Additional Owners that are being added to the group use the below-provided cmdlet. This cmdlet will set the value of Do not Notify at the time of adding new Additional Owners.

Set-Group -Identity "Group Name" -Add @{"AdditionalOwners" = "Name of the Additional Owner"} -NotifyOptOutAdditionalOwners "Name of the Additional Owner"

 

  1. To set Do Not Notify for already available additional owners, use the below-provided cmdlet.
Set-Group -Identity "Group Name" -Replace @{"AdditionalOwners" = "Name of the Additional Owner","Name of the Additional Owner"} -NotifyOptOutAdditionalOwners "Name of the Additional Owner"

 

  1. To bulk set Do Not Notify for already available Additional Owners, use the below-provided cmdlet. This cmdlet will set Do Not Notify for all the additional owners of a group.
$i=Get-Group -Identity "Group Name"| Select -ExpandProperty AdditionalOwner; foreach($j in $i.split(",")){Set-Group -Identity "Group Name" -Replace @{"AdditionalOwners" = $j,$j} -NotifyOptOutAdditionalOwners $j}
