How to set timeout of Threat Manager web console.

Summary: The Threat Manager web console's timeout is very short which will cause end users to have to constantly re-log in.

Issue: The Threat Manager web console is configured to time out after one minute.  This will cause a logged in user to have to re-enter their credentials after the one minute timeout.  This can become tedious for any user who needs access to the console, but may not be performing actions within constantly.

Instructions:

  1. Download PS script.
  2. Open script and set timeout value in minutes by editing the variable "$TimeOutMinutes"
  3. Run script on the Threat Manager server hosting the PostGreSQL database.
