Troubleshooting Scheduled Tasks

Issue: Scheduled Task Issues

for example, maybe the scheduled tasks does execute and says it ran successfully, but StealthAUDIT doesn't actually run the job.

Instructions: If role based access is enabled and the account on the scheduled task hasn't been assigned administrator rights in rba, the task will fail.  

Order of Operations 

1 - scheduled task executes
2 - SA.exe starts
3 - core logs get created
4 - connects to sql
5 - loads job tree

run w highest privileges checked on scheduled task

credential on scheduled task has access to sql if windows auth

Do Not Allow Storage of .NET Credentials policy needs to be off
