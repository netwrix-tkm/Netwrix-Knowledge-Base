Hotfix: SR 1.4 HF 002

Summary: This hotfix allows users to login as user@domain.com, domain\user, or domain.com\user as well as resolving an issue where a backup job would get stuck if the service was restarted while the job was running.

Issue: Issue: SR-5548 - Some backup/restore jobs get stuck in a running status if the server is restarted while the job is running
Resolution: Upon service start, all jobs that have a status of running are set to instead have a status of failed

Issue: SR-5525 - Users are only able to login using username@domain.com or domain.com\username, not domain\username
Resolution: Support was added to allow users to logon using domain\username
