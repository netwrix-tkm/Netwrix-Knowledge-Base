Issue:

Active Directory Inventory sometimes takes 200+ hours to complete. The usual run time in ~20 hours.

Every scan has numerous errors including:

Failed to parse Security Identifier. Trying another format.

 

System.NullReferenceException: Object reference not set to an instance of an object.

at Stealthbits.StealthAUDIT.DataCollectors.ADInventory2.Tasks.ADInventoryScanTask.InternalProcessRecord()

at Stealthbits.StealthAUDIT.DataCollectors.Tasks.Task.StartExecute()

 

access to the SQL server and the domain controllers did not indicate a connection issue during troubleshooting.

The null reference exception shows up as a consequence of the timeout, and is now fixed.  

 

 

Instructions:

  1. Unblock the hotfix zip file in the windows property dialog if an unblock button exists there.
  2. Close all instances of StealthAUDIT (check task manager under processes for all users)
  3. Exit StealthAUDIT console
  4. Copy and replace files from the folders in this hotfix to the corresponding folders in SA Install dir.
