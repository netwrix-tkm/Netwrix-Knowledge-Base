How To: Bulk Change the Object Types for Smart Groups Residing in an OU

Applies To:

Netwrix GroupID 11  GroupID Management Shell

Business Scenario:

In some environments, there may be a need to update the object types for multiple Smart Groups within an Organizational Unit (OU). For example, your current Smart Groups might include users, contacts, and groups in their membership criteria. However, due to a change in business requirements, you now need all Smart Groups in that OU to include only the following object types:

  • Users with Mailboxes

  • Users with External Email Addresses

  • Contacts with External Email Addresses

Manually updating the object types for each Smart Group can be time-consuming. This article explains how to apply the changes in bulk using the GroupID Management Shell.

Solution:

You can use a PowerShell cmdlet in the GroupID Management Shell to update the object types for all Smart Groups within a specific OU.

Note: The cmdlet may take time to execute depending on the number of Smart Groups in the OU. It is recommended to first test it on a smaller OU to validate results before applying it in a production environment.

Steps to Bulk Update Object Types:

  1. Launch the GroupID Management Shell.

  2. Log in using your service account.

  3. Run the following command:

     
    Get-SmartGroup -SearchContainer "Distinguished Name of Organizational Unit" -SmartFilter "(IMSGManagedGroupType=2)" | Set-SmartGroup -ObjectTypes "1","2","3"

     

     

  4. This command will update all Smart Groups in the specified OU to include:

    • Users with Mailboxes

    • Users with External Email Addresses

    • Contacts with External Email Addresses
