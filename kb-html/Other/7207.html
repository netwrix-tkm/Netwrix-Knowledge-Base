Access Analyzer Messages Bug Causes Job Messages to go to SADebug log

Summary: Sometimes job log messages may end up getting written to the SADebug log opposed to the job log.

Issue: Sometimes job log messages may end up getting written to the SADebug log opposed to the job log.

Instructions: When a data collectors query is modified in a job, it uses the application to write any messages to the SADebug log in case something goes wrong with the UI. If that same data collector is then used during job execution, it is possible that the data collectors' messages will continue to write to the SADebug log instead of the job log. Restarting Access Analyzer should make it so the data collectors messages will get written to the job log again.
