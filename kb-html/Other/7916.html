Hotfix: SA_8.2_056

Summary: Error getting scan database from execution host: Capacity must be positive. Parameter name: capacity

Issue: Summary:  Error getting scan database from execution host: Capacity must be positive. Parameter name: capacity
Before Fix: When copying large tier 2 files back from the proxy to the Access Analyzer console, an error will occur during decompression and the compressed tier 2s will be lost. (Tier 2s on the Access Analyzer console and proxy are not affected. The issue is the copy of the tier 2s that FSAA zips and tries to send back to Access Analyzer console.)
After fix: Tier 2s are copied back as they normally should.

Instructions:

  1. If you have received this error, delete tier 2s on proxy/remote server prior to applying hotfix
  2. Unblock the hotfix zip file in the windows property dialog, if an unblock button exists there. 
  3. Close all instances of Access Analyzer (check task manager under processes for all users)
  4. Update all DC files in your %SAInstallDir%DC folder
  5. Update all PrivateAssemblies in your %SAInstallDir%PrivateAssemblies folder.
  6. Run the 'Update Proxy Service' query under the Maintenance tab in the FSAA data collector against your proxy servers (if running proxy service).
