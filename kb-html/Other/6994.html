SIWinConsole_SIEntManager_SIWindowsAgent_5.2.0.226

Summary: Forged PAC possible false positives Detailed Summary of issue

Issue: Behavior before Fix:
Problem description:
It is possible that if user already has Kerberos:TGT ticket and then this user was removed from some group(s) then still old TGT ticket with old PAC will be used to get new Kerberos:TGS ticket. And in this case SI will detect this as Forged PAC attack. This can be interpreted as attack because even if user is removed from some group it still will use rights of this group to access network resources. But this generates too many attacks in SI which makes difficult to distinguish real attack from the case when user uses right from his old groups in situation when TGT ticket wasn't renewed.
Behavior after Fix:
Fix details:
Forged PAC attack detection usually interesting for 'sensitive' groups like Domain Admins, Schema Admins, and Enterprise Admins therefore new filter was implemented in Forged PAC analytics configuration which allows selecting particular groups (RIDs) for monitoring.
By default this filter includes Domain Admins, Schema Admins, and Enterprise Admins groups (RIDs) but user can extend this list by selecting other groups (RIDs) from Active Directory. If user will delete all entries from this filter then it will start collecting all RIDs as was in previous version.
Old Agents behavior: Older Agents will ignore this filter and will keep collecting events for all groups (RIDs).
