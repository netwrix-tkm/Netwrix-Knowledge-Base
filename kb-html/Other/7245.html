SI Agent - install registry keys

Summary: Steps to clean up incomplete SI Agent uninstall

Issue: In some cases an SI Agent may not have been fully uninstalled thus preventing re-install of the same or newer agent build.  Attempts to install the SI agent may report agent already installed.  This message is driven by detection / presence of specific registry keys.  This procedure can also be used if agent uninstall is 'broken' and user wants to remove it to re-install.
 

Instructions: First manually remove any remaining files from the previous SI Agent installation.  Default location is:

pre 7.x
"C:\Program Files (x86)\STEALTHbits\StealthINTERCEPT\SIWindowsAgent\"

7.x and newer

"C:\Program Files\STEALTHbits\StealthINTERCEPT\SIWindowsAgent\"

The SI Agent installers are built using either Wix for 5.x and later or Install Shield prior to 5.x.
To prevent a 'new' Agent installer from complaining that an existing agent is still present find and remove the following registry keys.  Depending on history of the machine one or all three keys may be present.

Wix version:
1. Key="HKLM\SOFTWARE\STEALTHbits Technologies\StealthINTERCEPT Windows Agent"
   Value="Installdir"

Install Shield version:
2. Key="HKLM\SOFTWARE\StealthINTERCEPT\Windows Agent"
   Value="InstallBase"
3. Key="HKLM\SOFTWARE\StealthINTERCEPT\Windows Agent"
   Value="ProductCode"

Run new agent installer which should no longer report that a previous agent version is present.
 

Product: StealthINTERCEPT
Module: SI - Windows Agent
Versions: Any agent prior to 7.0
