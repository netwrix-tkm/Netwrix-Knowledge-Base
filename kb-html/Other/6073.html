Hotfix: SIWindowsAgent_7.2.0.284_7.3.0.189

Summary:

SI-4452 60463 - Active Directory lockdown policy does not work on ntSecurityDescriptor

Behavior before Fix:

object classes were converted to lower case & comparison was case-sensitive, which would never evaluate comparison vs 'domainDNS' to true. Also evaluation was substring-based so 'domain' would evaluate to true vs 'domainDNS'.

Behavior after Fix:

object classes are not converted to lowercase, comparison is kept case-sensitive. Substring-based evaluation adjusted to either do full match, or search till ' |' (to cover case of multi-value objects classes)
