SA_FSAA_FileAges Table

The File Ages table contains information about the age of files within folders by looking at the created timestamp.   This table is generated by the File System Access Auditing primary data collection.   (FileSystem> 0.Collection> 1-FSAA System Scans, or equivalent.)  The text in red below is information that is not in the schema guide.
 
Columns
Name Type Description
HOST INTInteger ID of the file system host
ResourceID INTUnique ID of the share, folder, or file, references the SA_FSAA_Resources table
LastModified DATETIMEDate of the last time the resource was modified.  This is listed in UTC time.  This is calculated from ONLY files directly in the folder (i.e., those associated with the ResourceID in the same row), not files in subfolders. This column is based off the DATE MODIFIED time of the files in the folder.  The column is populated with the time of most recently modified file. (See also configuration requirements below.)
FileCount0 INTNumber of files within the folder that are between 0 and 1 year old  (See also configuration requirements below.)  The FileCount columns are based on the file's LAST MODIFIED date. 
FileCount1 INTNumber of files within the folder that are between 1 and 2 years old
FileCount2 INTNumber of files within the folder that are between 2 and 3 years old
FileCount3 INTNumber of files within the folder that are between 3 and 4 years old
FileCount4 INTNumber of files within the folder that are between 4 and 5 years old
FileCount5 INTNumber of files within the folder that are between 5 and 6 years old
FileCount6 INTNumber of files within the folder that are between 6 and 7 years old
FileCount7 INTNumber of files within the folder that are between 7 and 8 years old
FileCount8 INTNumber of files within the folder that are between 8 and 9 years old
FileCount9 INTNumber of files within the folder that are between 9 and 10 years old
FileCount10 INTNumber of files within the folder that are 10 years old or older

Instructions: Configuration requirements
 
To collect the file age breakdown (FileCount columns in the table), you must select "scan for file ages" in the FSAA query configuration.



NOTE: These settings will increase the time the scan takes to complete.  Additionally, more storage will be required on the Access Analyzer console (for SQLite DBs) and in the SQL Server database.
