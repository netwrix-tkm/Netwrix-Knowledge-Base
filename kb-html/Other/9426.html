msLAPS Events Are Not Omitting

(NOTE: This should be an employee only article)

Issue: Even though the omitproplist.txt file has a section for msLAPS events, there are some events still being collected by Netwrix Auditor.

Solution: There is a misspelling in the %INSTALLATION PATH%\Active Directory Auditing\omitproplist.txt file.  The following line needs to be corrected

*.msLAPS-EncryptedPasswordistory 

needs to be corrected to

*.msLAPS-EncryptedPasswordHistory

This is scheduled to be fixed in a future version of Netwrix Auditor.
