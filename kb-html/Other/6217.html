Hotfix: SIWindowsAgent_7.0.1.729_7.1.0.463_7.2.0.243

Summary:

I-7185 Investigate why there is no DNS host name for "Perpetrator: Host" for NTLM events

Behavior before Fix:

"Perpetrator: Host" value is filled with host name

Behavior after Fix:

A "Perpetrator: Host" value supports DNS names based on the config setting "hostResolutionDNS"

"hostResolutionDNS"  new option for SIWindowsAgent.exe.config file:

which is 'FALSE' by default and no changes in the agent behavior

When it is 'TRUE' the agent tries to resolve the value "Perpetrator: Host" as DNS name in case of the NTLM authentication event.
