Role Based Access (RBA) Error

Issue: StealthAUDIT opens one instance, but never starts up the application.
Upon starting a second instance, it will throw an error 'Role Based Access configuration was broken.'

Instructions:  Removing or modifying the configuration of RBA will remove all the existing Roles created.

All roles would need to be recreated after the below steps are followed.

In some instances; StealthAUDIT opens one instance, but never starts up the application (no splash page).
Upon starting a second instance, it will throw an error 'Role Based Access configuration was broken (The process cannot access the file 'path to PrivateAssemblies\rba-roles.conf' because it is being used by another process).
The issue is caused by an incorrect registry key for the StealthAUDIT event log. 
Navigate to and delete the registry folder hkeylocal and all subkeys if it exists:

machine/system/currentcontrolset/services/eventlog/application/StealthAUDIT 

1. On the occasion that there is a message:

'Role Based Access configuration was broken"

The REG_DWORD "RbaConfigured" can be deleted from the following Regkey:

HKEY_LOCAL_MACHINE\Software\Wow6432Node\Stealthbits\


2. In this instance the following file also needs to be deleted:

"rba.conf" file from the root of SA folder.

Do not remove the rba.config from the private assemblies folder!
