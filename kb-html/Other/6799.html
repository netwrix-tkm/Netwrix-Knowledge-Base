Issue: Summary:
This is a fix for SQL DC using the connection profile credentials to attempt to connect to StealthAUDIT database that the console is using.

Behavior Before Fix:
The logs indicate that there is an error using [connection profile username] to connect to [SMP database hostname] and access was denied.  The scan fails.

Behavior After Fix:
The SQL DC uses the system account to connect to the SA database
