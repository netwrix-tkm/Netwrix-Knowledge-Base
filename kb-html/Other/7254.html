SI - Better way to define Honey Pot Policy

Summary: SI Honey Pot Policy

 

Issue: Customer needed 'honey pot' policy so created an SI Policy which monitored a few services for changes to attributes badPasswordTime and badPwdCount.  While this could work it is not efficient as to make such a policy work one must also disable the global 'exclude noise events' pre filter for "AD User Logins with Bad Passwords".
 

Instructions: A better more efficient solution is to create an SI Policy with an Authentication monitoring registration with the desired service accounts selected in the "AD Perpetrator" filter and for the "Success" filter select to capture only failed authentications.  In this way the global 'exclude noise events' pre filter for "AD User Logins with Bad Passwords" can be left enabled to reduce that source of 'noise'.
