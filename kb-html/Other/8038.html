Hotfix: SA_8.2_062

Summary: Summary: This hotfix prevents an error from appearing when using angle brackets in Report names. Issue/Request: When using angle brackets in Report names, an "illegal characters in path" error would appear. Fix/Enhancement: Angle brackets are acceptable characters to use in Report names; no error appears.

Instructions:
1. Unblock the hotfix zip file in the windows property dialog if an unblock button exists there.
2. Close all instances of Access Analyzer (check task manager under processes for all users)
3. Exit Access Analyzer console
4. From an elevated command prompt, navigate to the Access Analyzer directory (default is C:\Program Files (x86)\STEALTHbits\StealthAUDIT) and run UnregisterAll.bat.
5. Copy and replace files from the folders in this hotfix to the corresponding folders in Access Analyzer Install dir.
