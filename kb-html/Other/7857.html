Summary: FSAA Exceptions Calculation Fails Because of Mathematical Overflow
Issue: Previously the FSAA_Exceptions job was reassigning IDs by incrementing the values starting after the max ID value of the previous run. That happened when new 'Stale User' exceptions were found. If the ExceptionID value reaches billions, the next run causes a mathematical overflow error and aborts processing because it maxes out the size on that column.
Fix: An analysis task to re-index the IDs was added to the DataAnalysisTasks.XML file of the 3-FSAA Exceptions job.
