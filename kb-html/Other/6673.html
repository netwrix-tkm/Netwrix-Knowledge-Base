Summary: FSAA - Only Update SA_FSAA_Hosts.LastScanTime at the end of a successful bulk import

Issue: Summary: FSAA - Only Update SA_FSAA_Hosts.LastScanTime at the end of a successful bulk import

Issue: SA_FSAA_Hosts.ScanTime was being updated even if the bulk import failed.

Fix: Now the DC will update that value only at the end of a successful Permissions bulk import. If the import is paused, the value won't update until the import is resumed and finishes successfully. Activity or Sensitive Data bulk imports won't update this value (only USN).
