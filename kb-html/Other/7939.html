Hotfix: SA_8.1_136

Summary: There was an issue in SPAA that was discovered due to a customer's onboarding process for switching contracted employees to full time. They were changing the account name in AD, which would then get picked up as a new account with the same SID. In turn this led to our SPAA imports failing because we expect the SID to be unique.

Issue: Summary:
There was an issue in SPAA that was discovered due to a customer's onboarding process for switching contracted employees to full time. They were changing the account name in AD, which would then get picked up as a new account with the same SID. In turn this led to our SPAA imports failing because we expect the SID to be unique.
 
Behavior After Fix:
SPAA imports no longer return this error, we provide a warning in the Job Log that a duplicate SID was found and we replace the dupe's SID with a value of "X-1-1703".
