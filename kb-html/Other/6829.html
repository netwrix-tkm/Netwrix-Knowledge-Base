30210 - Access Analyzer | Exchange 2016 SDD Issues (Autodiscover service)
When running a SDD scan on an exchange 2016 Server the connection fails with an error "The Autodiscover Service cannot be located" This is a WIn 2016, Exchange 2016 Cluster of 4 mailbox servers. We have connected using two different connection profiles one being Username@Domain (Steathaudit local) and one being an Active Directory Domain\User account. Host profiles have been tested are Excahnge Discovery query and a manual host list, also a host list with outlook.avntiplc.com (as per the auto discover URL) and this is the internal DNS for Autodiscovery.
