Hotfix: SA_9.0_009

Summary: Invalid column error was interrupting permissions scans against sql server 2008

Issue: Summary:
Invalid column error was interrupting permissions scans against sql server 2008

Behavior Before Fix:
When scanning sql 2008 servers the permissions scans would interrupt with this error: Failed to collect permissions: System.Data.SqlClient.SqlException (0x80131904): Invalid column name 'owning_principal_id'.
Invalid column name 'is_fixed_role'.

Behavior After Fix:
Permissions scans against sql 2008 servers should complete successfully
