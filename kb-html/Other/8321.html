Increase the number of IPs per target

Issue:

Error message when trying to create a target (greater than /22) at the GSM console

Cause:

  • All GSMs have a default limit of 4094 individual addresses per target.

Resolution:

  • Access the GOS administration menu and follow the instructions below:

    • Log in to the GSM as 'admin' via SSH or console

    • Go to the shell via 'Advanced > Support > Shell'

    • Set the new limit with these commands:

$ set max_ips_per_target 6000
$ save

The above example will allow entering up to 6000 IPs. The upper limit is 70000, default is 4096.

  • You can check the set value with:

$ get max_ips_per_target

  • Leave the shell with Ctrl+D (or 'exit')

Recommendation:

Warn the customer this can seriously impact the performance of the GSM when handling e.g. reports of densely populated /16 networks or running several of these scans in parallel. So if they experience performance hits we advise using smaller target ranges.
