Hotfix: SIWindowsAgent_4.0.0.672

Summary: AD Shared memory IPC overflow

Issue: Summary:  AD Shared memory IPC overflow
Before fix:  Agent Logs are overflowing for production domains.
After fix:
Introduced Client side Timeout for AD Search operations which are using DirectorySearcher class.
This timeout by default is set to 2 seconds but can be changed using "directorySearchTimeout" parameter in the config file.
 

Instructions: 1. Stop SIWindowsAgent service
2. Extract attached archive to the "C:\Program Files (x86)\STEALTHbits\StealthINTERCEPT\SIWindowsAgent" folder
3. Add the following line into section of SIWindowsAgent.exe.config file:

The format is HH:MM:SS (hours:minutes:seconds).
The default value is "00:00:02" - 2 seconds. So, if there is no such line in the config file then timeout will be set to 2 seconds.
 
4. Start SIWindowsAgent service
