Large SI LDAP Tables

Summary: If you have gigantic LDAP tables from an un/poorly scoped LDAP policy, you can truncate those tables.

Issue: The NVMonitorData db contains gigantic LDAP tables from an unscoped or poorly scoped LDAP policy, and running db maintenance takes more than days or weeks to run. 

Instructions: It is recommended to run database maintenance to remove this data, if feasible.  
In our current situation the customer's e_ldap table is 1.8 TB.  Database maintenance isn't exactly a feasible option.  

You can truncate the following 3 tables rather than running db maintenance for a long time.  
This will delete ALL LDAP Policy data.  


truncate table E_LDAP
truncate table E_LDAP_EventTracker
truncate table E_LDAP2Policy

Product: StealthINTERCEPT
Module: SI - Policy
Versions: versions with LDAP policy center
