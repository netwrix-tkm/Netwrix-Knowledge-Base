Configure SI to feed SI AD events to Access Analyzer via SAM

Summary: Feeding SI AD events to SAM using AMQP

Issue: In some environments SI is used for general purpose real time AD event collection and notification but customer also wants to send AD Events to Access Analyzer and DEFEND.  If need was for DEFEND only no problem as SI can send directly to DEFEND but what about Access Analyzer?  One approach is to let SI do the initial AD event collection then forward that data to SAM and use SAM to distribute the data to Access Analyzer, DEFEND etc.
