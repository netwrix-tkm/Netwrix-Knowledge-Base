OAM error "STATUS_ACCESS_DENIED - {Access Denied} "

For Windows 11 24H2, Microsoft has implemented a new Group Policy setting called Configure SAM change password RPC methods Policy. This new policy puts in place a new password change method required by default.


The workaround until a hotfix is available is to set the systems to be managed by NPS-D using OAM back to pre-patch behavior.  Setting this new policy to "Allow all"
 
This can be tested on a single Windows 11 24H2 system by changing this setting using the Local Group Policy Editor:
    1. Open "Edit Group Policy"
    2. Navigate to:  Local Computer Policy > Computer Configuration > Administrative Templates > System > Security Account Manager > Configure SAM change password RPC methods policy
    3. Enable and change the option to "Allow all change password RPC methods"
    4. Click the [Apply] and [Ok] buttons
 

 
This workaround can be applied via GPO to the NPS-D endpoints using OAM.  Either added to the same GPO that is used for the NPS-D scan service account  Remote SAM permission, or by creating a new GPO.
 
For Server 2025 GPO:
    1. Locate this option under: Computer Configuration > Policies > Administrative Templates > System > Security Account Manager > Configure SAM change password RPC methods policy
    2. Enable and change the option to "Allow all change password RPC methods"
    3. Click the [Apply] and [Ok] buttons
Note: When a hotfix is release, this setting requirement or acceptable setting selection will change.
 
For prior versions of Server GPO, the option does not exist, so we will need to push the setting out via a registry update:
    1. Navigate to Computer Configuration > Preferences > Windows Settings
  2. Right-click on Registry and click New > Registry Item
  3. Enter the following data into the corresponding fields. See screenshot example below:
        Hive: HKEY_LOCAL_MACHINE
        Key path: SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System\SAM
        Value name: SamrChangeUserPasswordApiPolicy
        Value type: REG_DWORD
        Value data: 3
    4. Click the [Apply] and [Ok] buttons
Note: When a hotfix is released, this Registry Item can be removed from the GPO.
