Support Process - Software Engagement Approach

Netwrix Support's Software Engagement Approach

Whether working with a client who has been a customer for years or conducting the first session of a proof of concept, there are a number of key features which represent the core of the Change tracker product and indorse the messages communicated by the sales team. A brief summary of the sales messages describing how the software aids an organisations cybersecurity posture are detailed here.

Sales Messages Summary

Make sure systems are fit for purpose - Assess a system's vulnerability. This should be priority number one, make sure that all systems are fit for purpose by removing all vulnerabilities. Prevention is much more desired than capture, there is little point installing cctv cameras in your home if all the doors and windows are unlocked and opened. (Feel free to use your own example!)
Handle change within the environment - Once systems are secure, monitor their configuration to identify change which could introduce vulnerabilities and monitor the file system to detect malicious activity. Monitoring systems for change is very much a strength in depth approach. AV is a must for recognising and eliminating known threats, if a threat is unknown to AV or a zero day attack then a file integrity solution is essential to alert organisations to unexpected changes.
Provide context to the change - It is important that when changes are identified, that you are able to provide the context of why there was a change. Did this change relate to a patch or application upgrade?

To be Configured, Demonstrated or Explained in All Webex

It is important that all employees of Netwrix align with these messages. As support representatives we are arguably the best placed to communicate and expand the messages by ensuring, during each WebEx interaction with either prospect or customer, that the below features are explaining.

Demonstrate CIS Compliance reports

Netwrix are advocates of prevention rather than cure! Demonstrate the CIS compliance reports and how they ensure a system is fit for purpose. Identify a couple of rules which you feel comfortable explaining and show the remediation steps contained within the report.

Explain the Netwrix Remediation Kits

Hand in glove with the compliance reports are the Netwrix Remediation Kits. Available for both Windows via GPO or UNIX/Linux via scripts, these kits which are available to customers only (don't provide them in the POC) enable system to be quickly configured to match the suggested secure configuration of a compliance report.

Demonstrate close loop intelligent change control

Setup a decent amount of CLICC planned changes to demonstrate the different CLICC configurations. Show the pre and post change configuration and describe how this is providing the context of the change. "We have recorded the changes generated by an application update. Now if we apply these changes to all system which will accept this application update, Change Tracker will automatically recognise the changes as good known changes."

Explain the Events Action button

From the events page run the prospect through the event action options, describing each one and how they subtly differ to meet different planned change scenarios.
When setting up planned changes be deliberately general with the specifics of what the planned change will match and also generous with the time window. Inevitably once a prospect is left alone to test the product, they will make changes and we would prefer that some of these changes are identified as planned.

Explain the breadth of the Custom planned change

To the uninitiated our custom planned change filter will look complex so spend some time explaining this feature to customers. In a POC it would be best to try not to get drawn into setting up examples. It should suffice to say that any parameter which Change Tracker tracks can be used within the custom filter. Provide the below as explanations:
"We can monitor an important log file which will change often but only create an unplanned change if the size reduces. The implication here is that potential data has been removed from the file."
"Changes made by a specific user, we would not advocate this but is works as an example of what is possible. If all changes during a patch run will be installed from the same service account, these changes can be waved through as planned changes. Any changes made by another account will appear as unplanned."
If setting up as an example is unavoidable then I have put all the current examples in a separate article.

Netwrix FAST Cloud

Take every opportunity to talk about FAST Cloud with existing customers.
"An Netwrix cloud based offering which leverages continually updated reputational intelligence from leading manufactures to build an all-encompassing whitelist of known good and bad files. Change Tracker can query this whitelist to automatically investigate changes, drastically reducing the change noise which is the challenge for all security professional."
