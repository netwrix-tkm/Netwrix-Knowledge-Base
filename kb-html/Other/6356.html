KB Type: Bug

 

Issue:

MySQL (AWS) - skipping databases during SDD scan with error 'Object reference not set to an instance of an object.'

Prevents SDD from detecting data because it is skipping many things this way.

Example error from log:

01/12/2020 16:22:49 WARNING SQL Stealthbits.StealthAUDIT.DataCollectors.SQL.Providers.TaskBase.InternalProcessRecord "'MySqlSensitiveDataCollector' encountered an error on database 'demo_approval_service'. It will be skipped. Message: 'Object reference not set to an instance of an object.'" BOKF-1111-TST-US-EAST-2-RDSI-CDPINSTANCE2.CZP0ZVQ4K8UA.US-EAST-2.RDS.AMAZONAWS.COM DCExtract 1488 4652 1208?

Instructions:

  1. Unblock the hotfix zip file in the windows property dialog if an unblock button exists there.
  2. Close all instances of StealthAUDIT (check task manager under processes for all users)
  3. Exit StealthAUDIT console
  4. Copy and replace files from the folders in this hotfix to the corresponding folders in SA Install dir.
