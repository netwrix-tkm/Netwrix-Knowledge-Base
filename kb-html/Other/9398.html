Unable to launch the NTP console after the upgrade

The customer was on SI version SI 7.3 and upgraded to NTP version 7.4.

After the upgrade, the customer was unable to launch the NTP console. When launching the console, it shows the sign of loading for a few minutes and then gives the error message "Unable to communicate with the Netwrix Threat Prevention Enterprise Manager. Please check the configuration and start the service."

We discovered that the certificates in the cert info folder were too old, which was why we could not launch the application.

Â 

1- Make a copy of the Certsinfo folder (Just for backup)
2- Deleted all certificates from the certsinfo folder
3- Restarted the service for Netwrix Enterprise Manager

Repeat steps 1 & 2 for each agent
1- Logged in to the server where the agent is installed
2- Navigate through the SIWindowsAgent and open the Certsinfo folder
3- We deleted all certificates from the certsinfo folder on the agent side.

Launch the console for NTP and push the agent
