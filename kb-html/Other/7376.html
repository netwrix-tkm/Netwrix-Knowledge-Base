Generate Domain user File Activity

Summary: Automate Domain user activity in Lab environment

Issue: In Lab environments of FAM it is often difficult to simulate user activity events on a regular basis to report on.  In order to enrich the reports within StealthAUDIT, Activity data from multiple users needs to be generated.

Instructions: To resolve this issue create two PowerShell scripts using the below codes.
 
Pre-requisites:  Network/Local Share that ALL AD Users have access to
                                All AD users need to be able to execute the script (logon as Batch User rights Assignment) on the server the script is to execute from.
                                PowerShell executionpolicy set to unrestricted if not already done so.
 
Script 1:  This script has a series of variables which need to be set per environment (listed below).  The first script collects a list of AD Users from a pre specified OU.  It then randomly selects a user from the list and launches PowerShell as that user calling the second PowerShell Script.

$UserOU - set this to the OU where your user accounts are 
   $Path - Path to create the files can be UNC
   $MovePath - Path to Move files to, can be UNC
   $MaxfilestoCreate - Defines the max number of files to create. Note the second script will randomise between this Value.                                     
   $LogfilePath - where to generate the log for each job.  Log will be in datetime format                     

https://downloads.stealthbits.com/access/files/StealthTOOLS/AUDIT/script1.ps1

Script 2: This script generates a random list of files to create using different file extensions.  It will select a random number of files to create between 1 and the maximum number set in script 1.  The sequence of Activity events it creates are as below. 
 
Creates file > Modify file > Rename > Copy > Delete          
 
Between each event type there is a pause which is randomised between 1 and 120 seconds
