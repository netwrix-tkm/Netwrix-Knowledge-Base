Netwrix Threat Prevention and Activity Monitor for AD has dropped support for Windows Server 2012 R2

Have you purchased extended support from Microsoft for Windows Server 2012 R2 for this DC (domain controller)? (This is a critical requirement for starters.)
Is it only 1 (one) single domain controller that you are seeking to run the NTP agent on?
How many Windows Server 2012 R2 domain controllers are you seeking to run the NTP agent on?

Netwrix Threat Prevention and Activity Monitor for AD has dropped support for Windows Server 2012 R2.

1) Microsoft ended normal support for Windows Server 2012 R2. Security patches delivered as part of extended support are not shared by Microsoft to partners. This means Netwrix is unable to maintain support for Microsoft Windows 2012 R2 domain controllers.

2) You may elect to roll back to an older agent on your WS2012 R2 Domain Controller but you must understand the related risks.

3) There are known vulnerabilities in Threat Prevention 7.3.9 and earlier. These are addressed in NTP 7.4 and NTP 7.5. NTP Server 7.4 and 7.5 are not compatible with earlier agents.

4) Running WS2012 R2 compatible agents requires the use of a version of NTP with known vulnerabilities.

5) Any updates made by Microsoft to WS2012 R2, specifically LSASS, will not be accounted for in any version of NTP agent and may result in missed event capture.

6) You need to have a plan in place to replace all WS2012 R2 hosts.

If you have experienced crashes on WS2012 R2 for SI/NTP Agents please upgrade to following versions:

7.4.0.201
7.3.9.286

I am sorry we are not able to provide a more satisfactory solution, however please understand we evaluated many options to continue to support your scenario.
