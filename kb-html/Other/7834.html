Issue:
ADI Extended attributes truncation error seen by customer because of extended attributes being to large.
 

Instructions:

  1. Unblock the hotfix zip file in the windows property dialog, if an unblock button exists there. 
  2. Close all instances of StealthAUDIT (check task manager under processes for all users)
  3. Update DCADInventory.dll file in your %SAInstallDir%DC folder
  4. Update Stealthbits.StealthAUDIT.DataCollectors.ADInventory.dll file in your %SAInstallDir%PrivateAssemblies folder.
  5. In SQL Management Studio click tools > Options > Designers.  Uncheck "Prevent saving changes that require table re-creation"
  6. In SQL management Studio update table dbo.SA-ADinventroy_AttributeChange columns "OldValue" and "NewValue" to use nvarchar(MAX) instead of nvarchar(1024)
  7. In SQL management Studio update table dbo.SA-ADinventroy_ExtendedAttributes column "Value" to use nvarchar(MAX) instead of nvarchar(1024)
