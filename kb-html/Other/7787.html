Hotfix: SA_9.0_022

Summary: FSAA Limit maximum number of Files Setting not saving

Issue: Summary:  FSAA Limit maximum number of Files Setting not saving
Before Fix: Setting a limit to the number of file types to return on the 'File Properties' page was being saved in the xml but was not being loaded into the UI next time it was opened.
After fix: Configured setting is loaded into the UI next time it is opened.

THIS HOTFIX HAS A PREREQUISITE OF HOTFIX SA_9.0_018. PLEASE APPLY THAT HOTFIX PRIOR TO APPLYING THIS ONE. SEE KB FOR THAT HOTFIX FOR MORE DETAILS
Instructions:
  1. Unblock the hotfix zip file in the windows property dialog, if an unblock button exists there. 
  2. Close all instances of StealthAUDIT (check task manager under processes for all users)
  3. Update all DC files in your %SAInstallDir%DC folder
  4. Update all PrivateAssemblies in your %SAInstallDir%PrivateAssemblies folder.
  5. Run the 'Update Proxy Service' query under the Maintenance tab in the FSAA data collector against your proxy servers (if running proxy service).
