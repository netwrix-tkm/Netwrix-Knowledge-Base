Summary: Investigations for group changes has millions of events in queue

Issue: Summary: Investigations for group changes has millions of events in queue
Behavior before Fix:
Enterprise Manager sends SIEM messages for Events asynchronously. There is a dedicated in-memory queue for this. This queue has two limits: syslogMessageQueueSize (default value is 100 000) and syslogMessageQueueThreshold (default value is 30 000).
syslogMessageQueueSize - is a limit for total number of entries in the queue.
syslogMessageQueueThreshold - is used to decide how to put item into the queue using the following logic:
if number of Event OBJECTs in the queue is less than syslogMessageQueueThreshold then put into the queue entire Event OBJECT, otherwise put into the queue only Event ID which will be used later to query Event data from the Database.
Previously logic to calculate number of Event OBJECTs in the queue was implemented by going through entire queue and counting such items which caused delays on Event enqueue.
Behavior after Fix:
Now logic is updated to have separate counter for such items which is incremented every time when object is enqueued and decremented when object is dequeued.
