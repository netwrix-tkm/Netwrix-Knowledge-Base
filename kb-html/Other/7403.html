SAM NETAPP C-Mode configuration for multiple NICs

Summary: Specifying correct IP address for FPolicy in SAM configuration when Agent host has multiple NICs

Issue: The autoconfiguration feature ("Configure FPolicy") for C-Mode NetApp needs an IP address of the agent's machine to register in NetApp. In particular, the IP address is specified in the External Engine FPolicy object. NetApp nodes use this IP address to connect to the SFAM Agent.

When the agent's machine has more than one network adapter, the autoconfiguration feature fails with the following error in FPolicyServerSvc.log file:
Multiple local IP addresses found. Set the FPOLICY_SERVER_IP parameter.
 

Instructions: The autoconfiguration feature ("Configure FPolicy") for C-Mode NetApp needs an IP address of the agent's machine to register in NetApp. In particular, the IP address is specified in the External Engine FPolicy object. NetApp nodes use this IP address to connect to the SFAM Agent.

When the agent's machine has more than one network adapter, the autoconfiguration feature fails with the following error in FPolicyServerSvc.log file:
Multiple local IP addresses found. Set the FPOLICY_SERVER_IP parameter.

Instructions: 
1.  Open up the SBTFileMon.ini under "Program Files\STEALTHbits\StealthAUDIT\FSAC". 

2.  For each Monitored host in the SBTFileMon.ini file, add the following line:

      fpolicy_server_ip=''

 3.  Restart agent service.
