Hotfix: SIEntManager_SIWinConsole_3.4.2.137_SI-3177_I-1338

Summary: Improve handling of 'NULL' values in tables for Dynamic collections

Issue: Before fix:
. table for Dynamic collection with NULL value may lead to 'SqlNullValueException: Data is Null' exceptions
. A lot of warnings [Dynamic collection tables checksum checker] WARN DynamicCollectionsResolver [(null)]
. Console: Selecting the Dynamic table with 'NULL' value leads to the System.FormatException

After fix:
. 'SqlNullValueException: Data is Null' exception is replace by the 'ERROR DynamicCollectionsStorage [(null)] - SelectPerpetrators failed System.ArgumentNullException: Value cannot be null'
. Warning will be displayed only if lockdown perpetrator in dynamic collection table has empty sid cause sid is essential for lockdown rules.
