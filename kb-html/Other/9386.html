Why my Form change log is showing so many data in its Diff section?

Custom Forms in NetSuite are customizations that often get their XML definition modified because of other changes that were actually made somewhere else in the system.

Resulting changes to the custom forms object definition do not leave any traceability at the custom form level, neither as system notes, history or audit trail information.

It is common then to find form change logs that contain lots of data because once a direct modification of the form is detected our system scans its definition and the resulting diff is the one compared to the last documented definition which usually includes changes that were applied to the object from an external source.

These are some examples:

  1. A new Custom Entity Field is created in NetSuite and is set to be applied on Customer and Vendor records.
    In this case as soon as the field is saved, all Customer and Vendor forms in the system are updated to include the new field as part of their configuration.
  2. Sublist applied to Customer records
    In this case the change affects all customer forms in the account but the form itself was not directly modified neither it shows any activity in its History or System Notes
  3. Subtab created for Transaction standard record type
    As with the previous examples, the tab would be available on all Transaction Forms in the account without the form actually being modified

Bundles and SuiteApps also create, modify or remove the objects mentioned above adding more changes to the forms present in the account.
