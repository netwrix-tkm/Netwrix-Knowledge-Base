Powershell Error - cmdlet not found

Issue: The term 'get-powershellcommand 'is not recognized as the name of a cmdlet, function, script file, or operable program. Check the spelling of the name, or if a path was included, verify that the path is correct and try again.

Instructions: This could happen with ExchangePS, PowerShell or Exchange2k DC.  It usually means the appropriate cmdlet you are trying to run is not available.  Check that the Management Tools that allow access to that cmdlet are installed and that PowerShell itself is installed.  If they are installed, it could mean that the account trying to run the command does not have the appropriate rights to access that particular cmdlet.  Test the same command in native PowerShell adding the snap-in or in Management Shell for the product that cmdlet is associated to.
