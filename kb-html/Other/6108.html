Hotfix: SA_10.0_092

Summary:  SP Broken Inheritance Analysis: Add 'DROP VIEW' statement to avoid conflict with table

Before Fix: Performance improvements made to the job changed one of the objects we created from a view to a table with the same name. When upgrading from the older version of the job to the newer one, a conflict would occur because the pre-existing view in the older version of the job was never dropped.

After Fix: Modified analysis to drop the old view before creating the new table with the same name.
