Summary: Agents not collecting events and "Member 'fqdnAgents' was not found" errors observed after upgrade to 5.2.0.266

Issue: Summary: Agents not collecting events and "Member 'fqdnAgents' was not found" errors observed after upgrade to 5.2.0.266
Behavior before Fix:
When new feature was implemented which allows to select between FQDN and NetBIOS name for HOST parameter in the 'SBTFileMon.ini' file for 'File System StealthAUDIT' event type it was needed to add new parameter ('fqdnAgent') into policy filter which contains a list of Agents where HOST must be in FQDN format. Therefore if new Agent is running under old EM it can't find this parameter and generates error.
Behavior after Fix:
Now behavior in Agent changed to always use NetBIOS format if there is no 'fqdnAgents' parameter in the filter.
