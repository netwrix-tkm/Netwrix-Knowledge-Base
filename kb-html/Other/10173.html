How To: Set delivery restrictions on dynasty groups

Applies To:

Directory Manager 11

Business Scenario:

Is there a way to assign/restrict specific users from sending emails to a Dynasty?

More Information:

A Dynasty is a Smart Group that creates and manages other Smart Groups using information in the directory. Dynasties help you manage large distribution lists by creating hierarchical group structures that represent your organization. The Smart Groups that the Dynasty creates are called child groups and become members of their respective parent Dynasty.

Steps:

Using the Directory Manager, it is possible to restrict the emails sent to a Dynasty. 

This requirement can be achieved by doing the following:

1. In the Directory Manager application. Expand Groups->All Groups->Dynasties 

 

 

2. Select any middle Dynasty, for example "Direct Reports of Abbey.Crawford", on which you would like to set the delivery restriction, and go to its Properties.

3. Click on the Delivery restriction tab, and you will see 3 types of filters, which control who can and cannot send email messages to a group/Dynasty.

    • Quick Filter (default filter)
    • Accept Messages (to allow all or specific users/groups to send emails to the Dynasty)
    • Reject Messages (to restrict specific users/groups from sending emails to the Dynasty)

4. In the Accept Messages from Click on Add. Now add any user who can send emails to the Dynasty. 

 

5.  After adding the user click Save.

These steps can be followed to allow/restrict any specific user or group to send emails to a Dynasty.

Points to remember:

Before performing the steps given above, make sure that the following properties are present:

1. On the Dynasty Properties , click the Dynasty Options tab. 

2. In the Dynasty Options, make sure that the Always Inherit option is selected.

 

 

3. Next, please go to the Admin center->Go to the settings of the identity store. 

4. Click on configurations tab->Now click on the Dynasties tab. 

5. Make sure authOrig and unauthOrig are present under Selected Attributes. These attributes control authentication, which is vital to apply a restriction or rule to Dynasties, like the scenario discussed in this article.
