Extend the NTM Web console CommandTimeout

Symptom:

Queries beyond the default one hour time out.

Cause:

Timeout Settings to low for the volume of events processed in the environment.

Resolution:

If not on the latest Builds of NTM and Postgres Patch the Server first.


Adjusted a time out "CommandTimeout": 30,  in the *\STEALTHbits\StealthDEFEND\WebService\appsettings.common.json"
to 900.

Restart the Related Service. Netwrix Threat Manager Web Service   (SBTSDWEB)

Or within an Admin PowerShell window.

## Restart the Netwrix Threat Manager Web Service

Restart-Service SBTSDWEB -Force

## Check all NTM services post Restart

Get-Service -DisplayName "Netwrix Threat Manager*" |ft -AutoSize

 

Or reboot the server.
