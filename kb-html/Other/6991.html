Hotfix: SD_1.1_006

Summary: StealthDEFEND not processing High Risk Permissions

Issue: Instructions to resolve issue: 27086 - Stealth Defend not processing High Risk Permissions
Attached is a ZIP file containing a python file "open_access.py" This is a "patched" version of the python job that evaluates events for Dangerous Permissions.

The issue you have encountered was resolved in our 1.2 Version (releasing in August 2018) so we have simply delivered this patched file to resolve the issue.
 

Instructions: Installing the Patch

  1. Extract the "open_access.py" file in the attached ZIP file into "C:\Program files\STEALTHbits\StealthDEFEND\JobService\Python\Jobs" directory and overwrite the existing python file. This will "install" the new patched version of High Risk Permissions.
 
  1. Next you should restart the Windows Service named "StealthDEFEND Job Manager" - this will ensure the updated job is running.
 
At this point your backlogged High Risk Permission events should be evaluated and new threats will be created for any threats found in the backlogged events. It may take a few minutes for the backlogged high risk permission queue to start decreasing.
