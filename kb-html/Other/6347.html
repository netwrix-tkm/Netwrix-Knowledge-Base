Hotfix: SbPAM 3.0sp1 CU2

Summary:

This CU addresses multiple issues:

LAPS activities are not using the Administrator password and RDP sessions will not connect. Post session logoff action are not logging off a connected Administrator.

Users are not getting removed from groups post session.

DUO SAML authentication is not able to be configured. When looking in the logs there is an error about a bad date format.

Adding > 100 resources to an access policy causes the Session wizard to load for about a minute before letting the user select the activity.
