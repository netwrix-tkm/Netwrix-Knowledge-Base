Hotfix: SIWindowsAgent_7.2.0.216

Summary:

SI block unconstraint delegation policy set only for users class but blocking computer and user class objects

Behavior before Fix:

This happened because computer class is inherited from the user class. And current implementation of blocking is respecting class inheritance

Behavior after Fix:

To add more flexibility for blocking we decided that in this particular place we don't want to analyze inheritance anymore and will block only on the top class name of the object. So, the code was modified to respect only top class name for AD Blocking and ignore class inheritance.
