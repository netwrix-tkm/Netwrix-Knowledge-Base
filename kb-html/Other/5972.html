How to change your Threat Manager PostgreSQL password using the Threat Manager database tool.

Issue: Prior to the creation of the database tool, it was not possible for Threat Manager users to change their default PostgreSQL password.

Prerequisites:  .Net 5.0 required to use application, if it is not installed user will be prompted to install it.

 

Instructions:  

  1. Download the StealthDEFEND_DatabaseTool
    1. https://downloads.stealthbits.com/access/files/SDHotfixes/Tools/StealthDEFEND_DatabaseTool.zip
  2. Navigate to the StealthDEFEND_DatabaseTool folder
  3. Locate the Threat Manager Database Tool.exe executable file and run it
  4. Contact Netwrix support for the password to use the Database Tool

 

Changing a password

  1. Navigate to the Change Password tab
  2. Enter the new password
  3. Click Change Password
  4. Navigate to the Restart Services tab and click Restart Threat Manager services for your changes to take effect

 

Creating a new user

  1. Navigate to the Create New User tab
  2. Enter information for all fields
  3. Click Create User

 

Changing the config user

Note: The user drop down will be empty if no new users have been created

  1. Navigate to the Change Config User tab
  2. Select the user from the dropdown list that you would like to be the new config user.
  3. Enter the users password
  4. Click Set as Default
  5. Navigate to the Restart Services tab
  6. Click the Restart Threat Manager Services button

 

 

 

 

Disabling a user

Note: The user drop down will be empty if no new users have been created. This only shows enabled accounts that can be disabled.

 

  1. Navigate to the Disable User tab
  2. Select the user from the drop-down menu that you would like to disable
  3. Click Disable User

 

Enabling a user

Note: The user drop down will be empty if no users have been disabled. This only shows disabled accounts that can be re-enabled.

 

  1. Navigate to the Enable User tab
  2. Select the user from the drop-down menu that you would like to enable
  3. Click Enable User

 

Restarting Services

  1. Navigate to the Restart Services tab
  2. Click Restart StealthDEFEND Services
  3. Allow a minute for the services to restart
