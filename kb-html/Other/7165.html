maximum concurrent shells has been exceeeded

Summary: Exchange PS (maybe powershell) errors out with "maximum concurrent shells has been exceeeded".

Issue: Exchange PS (maybe powershell) errors out with "maximum concurrent shells has been exceeeded".

Instructions: You can run this command to find out how many concurrent shells are allowed.  
get-item wsman:\localhost\shell\maxshellsperuser

However, in our case they were set to 30 and the following resolved the issue: 

A developer mentioned that "WinRM can easily get into states which require machine reboot to clear" and recommended these steps on the CAS server: 
1) recycle the IIS app pool which runs the powershell endpoint
2) reboot the server

Feel free to try the first before the second, if you'd like to not reboot the server.  I would recommend rebooting the server, though.
