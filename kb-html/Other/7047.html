Summary: SI Exchange events in Splunk issue after upgrade

Issue: Summary: SI Exchange events in Splunk issue after upgrade
Behavior before Fix:
In this place Splunk template uses combination of 'Event: Type' and 'Event: Name' fields. When code was re-factored in 6.0 to support 'server-less' mode for SAM integration the part which is responsible for filling 'Event: Name' field with the value from 'Event: Raw Name' in case of empty value for several event types was missing. Therefore 'Event: Name' become empty.
Behavior after Fix:
Agent code updated to fill 'Event: Name' field with the value from 'Event: Raw Name' in case of empty value for all supported event types.
