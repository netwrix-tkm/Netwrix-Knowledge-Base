Tail a log file in Windows

Summary: How to tail a log file in Windows

Issue: Sometimes its pertinent to tail a log file. Linux has this function natively from the command line, now Windows does too (Since PowerShell version 3)! This opens the log file as a system process so can be read while StealthAUDIT is writing to the file.

Instructions: Open a PowerShell prompt. Move to the log file directory by using the Change Directory command "CD". Once in the directory with the log file run  Get-content -Wait -Tail X Logfilenname.tsv Where X is the number of lines you want to tail (note you can use TAB to autocomplete these steps). (Using the -wait switch keeps the file open, failure to add the -wait will only display the current state of the log file).
 
E.g., to view the TSV for the Active Directory Inventory job, run the following.
Open PowerShell
CD "C:\Program Files (x86)\STEALTHbits\StealthAUDIT\Jobs\GROUP_.Active Directory Inventory\JOB_1-AD_Scan\Output"
Get-content -Wait -Tail 10 1-AD_Scan_Log.tsv will show you the last 10 lines of the AD Scan Job and then any new lines of the log file.
The output is refreshed every second.
 
Reference link - https://docs.microsoft.com/en-us/powershell/module/microsoft.powershell.management/get-content?view=powershell-6
