Hotfix: AIC_7.1_SP1_004

Summary: Ticket: E-mail notification and user being assigned to resource is being completed before changes in wizard are finalized Issue: When making membership changes through the Change Access wizard, the changes are normally committed in the Commit page, but the 'Previous' button was still enabled allowing the user to go back and forth in the wizard, this action caused emails to be sent to the modified member every time the next button was hit. Fix: - The 'Previous' button is now disabled in the 'Commit' page, so the user can't go back in the wizard at this point. - Also, in the 'Add Notes' pages the text "Click next to commit your changes" was added, to inform that the changes will take effect after hitting the 'Next' button.
