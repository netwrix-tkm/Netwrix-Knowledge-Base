How To: Add a new AD attribute to the Directory Manager replication schema

Description: This article helps you replicate custom AD attributes to the Elasticsearch repository.

Keywords: AD attributes, Custom Attributes, Replication, Active Directory, AD, Elasticsearch

Applies To: 

Directory Manager 11

Business Scenario: 

  1. We have defined a new custom attribute in Active Directory (AD), and we want to use that with Directory Manager.
  2. Or there is an attribute that we dont see in the Smart Group criteria

Add a New Attribute: 

  1. Lets assume that you have a new custom attribute or an existing AD Attribute that you do not see in Directory Manager (not in roles, filters, or smart group criteria).
  2. Just logon to the Directory Manager Admin Centre and navigate to the Identity Stores tab from the navigation pane. Click on options (3 dots) and then Edit the required Identity Store.

Figure 1 - Options Menu for Identity Store - Directory Manager 11

  1. In case of a new custom attribute, from the Settings pane in the middle, look Schedules and find a Schedule called Schema Replication, from its options (3 dots) and Run it.

Figure 2 - Running the Schema Replication Schedule - Directory Manager 11

  1. Wait for a few minutes (5-10), then find the Replication tab in the settings pane. And click on Add Replication Attributes:

Figure 3 - Add Replication Attributes - Replication Settings in Directory Manager 11

  1. On the new prompt, search for your new or missing attribute and select it, then hit the save button.

Figure 4 - Searching for Attributes - Replication Settings in Directory Manager 11

  1. Then you will see a prompt to Replicate objects, hit yes.

Figure 5 - Prompt to Replicate Objects - Directory Manager 11

  1. At the end, click on the Save button on bottom right of the screen to save these changes.

Figure 6 - Saving the unsaved changes - Directory Manager 11
