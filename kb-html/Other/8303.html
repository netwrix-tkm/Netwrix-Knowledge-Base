Mongo Commands - Change Tracker Poor Performance Prior to Version 7.2.1.20

If Change Tracker is running slowly or you find that the Change Tracker reports are running all the time then the system might be impacted by reports running against the report schedule used pre R2. If the old schedule is in use it is not visible through the GUI, and would require a mongo command to be revealed. Regardless, it is a prudent step if the Change Tracker version is prior to version .20 that the following command is run against the system.


Enter the following command: cd "C:\Program Files\NNT Change Tracker Suite\Gen7\MongoDB\bin"

This is the default location of the Mongo binary files. If you are unsure to which path the Mongo database has been installed, view the properties of the Mongod service and the path to execute parameter.

Enter the following command: mongo.exe

Enter the following command: use NNTHubService

Enter the following command: db.ScheduledPolicy.remove({_t:"MongoPolicyScheduleItem","TemplateType":2})

Enter the following command:Â db.AgentTasks.remove({"AgentTask.AgentTaskType":5});

The above commands will correct any occurance of the old style schedule (db.ScheduledPolicy.remove) and remove any reports that are waiting to be executed (db.AgentTasks.remove).
