Agent Upgrade in Netwrix Threat Prevention 7.3.5

Question

How to upgrade the Netwrix Threat Prevention 7.3.5 agent using a script?

Answer

A PowerShell wrapper to assist with SI Agent installations and upgrades is attached to this article.

Most parameters are true\false switches that default to false, with some parameters defaulting to true (-DoNotStartAgentService, -NoFirewallRules, and -NoDNSLookups). The -Features parameter is mandatory and tab completes to WindowsFile, ActiveDirectory and Exchange Â— the script will sort out the proper names.

Registering new agents

NOTE: Registering new agents requires a generated enrollment secret from the console.

Refer to the following code block for an example:

.\SIAgentInstaller.ps1 -EnterpriseManagerIP 10.0.0.51 -SIAgentExecutable 'C:\Users\da\Desktop\SI Agent.exe' -Log log.txt -Features ActiveDirectory, WindowsFile -EnrollmentSecret "0469 E33A 2091 C23F" -EMCertificate "E5475653ABF5907B197D1FD53F4963B55269D425"

Upgrading Agents

This can be truly unattended as the certificates are still in the left over CertsInfo folder. Refer to the following code block for an example:

.\SIAgentInstaller.ps1 -EnterpriseManagerIP 10.0.0.51 -SIAgentExecutable 'C:\Users\da\Desktop\SI Agent.exe' -Log log.txt -Features ActiveDirectory, WindowsFile
