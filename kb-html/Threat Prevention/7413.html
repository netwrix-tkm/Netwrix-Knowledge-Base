Cisco AMP V6 Affecting StealthINTERCEPT

Summary:  When Cisco AMP V6 is deployed StealthINTERCEPT deploys however silently fails

 

Issue:  When StealthINTERCEPT is deployed to a domain controller that has Cisco AMP V6.X installed SI will install OK and start the AD modules, however they will silently fail within a few minutes, the agent logs will show an error similar to the below:

2019-09-24 03:45:08,359 8232:[12] WARN  SI.SIMonitor.ClientServices.Master.MasterClient [(null)] - Register(), will retry: URI = tcp:3739/IMasterManager.tcp, socket exception: No connection could be made because the target machine actively refused it 10.50.122.16:3739, ErrorCode: 10061

2019-09-24 03:45:08,625 8232:[12] ERROR SI.SIMonitor.ClientServices.Master.MasterClient [(null)] - The agent failed to register

2019-09-24 03:45:10,687 8232:[12] WARN  SI.SIMonitor.ClientServices.Master.MasterClient [(null)] - GetPoliciesRevision(): URI = tcp://10.50.122.16:3739/IMasterManager.tcp, socket exception: No connection could be made because the target machine actively refused it 10.50.122.16:3739, ErrorCode: 10061

2019-09-24 03:45:12,750 8232:[12] WARN  SI.SIMonitor.ClientServices.Master.MasterClient [(null)] - GetCollectionsRevisionSet: URI = tcp://10.50.122.16:3739/IMasterManager.tcp, socket exception: No connection could be made because the target machine actively refused it 10.50.122.16:3739, ErrorCode: 10061

2019-09-24 03:45:14,812 8232:[12] WARN  SI.SIMonitor.ClientServices.Master.MasterClient [(null)] - GetLdapPoliciesRevision(): URI = tcp://10.50.122.16:3739/IMasterManager.tcp, socket exception: No connection could be made because the target machine actively refused it 10.50.122.16:3739, ErrorCode: 10061

 

Instructions:  Even if the client has used previous versions of Cisco AMP 5.X etc, they will need to re-add the exception for LSASS and Active Directory. The recommended exclusions are:

Executables: SIWindowsAgent.exe, SIGPOAgent.exe

SI 6.x File Paths: C:\Program Files (x86)\STEALTHbits\StealthINTERCEPT\SIWindowsAgent

SI 7.x File Paths: C:\Program Files\STEALTHbits\StealthINTERCEPT\SIWindowsAgent

Services:  SBTActiveDirectoryService, SBTLoggingSVC (If doing file logging also)

If upgrading agent from 6.x to 7.x ensure moved from Program Files (x86) to Program Files and configure AMP exclude path accordingly
