Server Backup Stuck in Queued Status

IMPORTANT: This article applies exclusively to Netwrix Recovery for Active Directory v2.6.

Symptom

In the Forest page in Netwrix Recovery for Active Directory, the Server Backup History table lists a queued session that does not start.

Causes

Review the list of possible causes:

  • A service connection point is not published.
  • The Recovery Server Backup Agent failed to install.
  • Netwrix Recovery for Active Directory encountered an issue during the backup process.
  • The Recovery Server port (9001 by default) is closed for inbound communication.
  • Your Netwrix Recovery for Active Directory license has expired.

Resolution

Follow these steps to remove the queued session from the list:

  1. Launch the Recovery Configuration Utility by double-clicking the Recovery_Config.exe executable located in the installation directory: ...Netwrix\Recovery for Active Directory\Recovery_Config.exe.
  2. Update the account password for SQL Server Authentication in the Login and Password fields.
  3. Run the NetwrixRecovery_Setup.exe executable as an administrator to launch the Recovery for Active Directory Setup wizard.
  4. On the Netwrix Recovery Server Configuration page, enter the port and credentials to be used for running the application server.
  5. In your SQL Server Management Studio instance, locate the Netwrix Recovery for Active Directory database. The default name is NetwrixRecovery.
  6. Right-click the NetwrixRecovery database and run a new query to output all records of the Server Backup History table:
    7. SELECT * FROM ServerBackupHistory
  7. Locate the entry with NULL in the EndTime column. Record the corresponding ID value.
  8. Run a new query to remove the queued entry:
    9. DELETE FROM ServerBackupHistory WHERE ID = '%targetID%'
  9. Refresh the Forest page in Netwrix Recovery for Active Directory to confirm the query is deleted.
