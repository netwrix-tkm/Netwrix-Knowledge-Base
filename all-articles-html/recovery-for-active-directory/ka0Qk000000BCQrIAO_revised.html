How to Migrate Netwrix Recovery for Active Directory v2.6 to a New Server

Overview

This article provides step-by-step instructions for migrating Netwrix Recovery for Active Directory (NRAD) v2.6 to a new server. The migration process ensures that all configurations and data are preserved while transitioning to a new environment.

Instructions

  1. Identify the location of the NRAD database:
    • Note the SQL Server name and Database name.
  2. Optionally, take screenshots of each configuration tab in the Netwrix Recovery Web Console for reference.
  3. Download the NetwrixRecovery_Setup.exe installer for v2.6 from the official Netwrix website.
  4. Verify that the installer file is unblocked before proceeding with the installation:
    • Right-click the file, select Properties, and ensure the Unblock checkbox is selected.
  5. Install NRAD v2.6 on the new server:
    1. Run the NetwrixRecovery_Setup.exe file.
    2. On the installation page, read and accept the End User License Agreement.
    3. During the installation process, connect to the existing NRAD database using the previously noted SQL Server and Database name.
  6. After the installation is complete, open the Netwrix Recovery Web Console on the new server.
  7. Optionally, compare the new configuration tabs with the screenshots taken earlier to ensure all settings are correctly migrated.
