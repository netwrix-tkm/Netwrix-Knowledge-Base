How to Reset Multi-Factor Authentication in Recovery for Active Directory

Question

Can you reset multi-factor authentication (MFA) in Netwrix Recovery for Active Directory (NRAD) after losing access to your authenticator account?

Answer

Yes, you can reset MFA in NRAD by following the steps below. After resetting, you can also re-enable MFA if needed.

Instructions

  1. Open SQL Server Management Studio and connect to the SQL Server instance used for NRAD.
  2. Expand the Tables in the database used by NRAD.

    NOTE: The default database name is NetwrixRecovery, but it may vary if modified during installation.

  3. Right-click the dbo.tusers table and select Edit Top 200 Rows.
  4. Locate the user account by identifying their login name in the Data column.
  5. Change the TwoFactorEnabled column value from True to False and save the changes.
  6. After resetting MFA, users can log in to the console without an authenticator code.

Re-Enabling MFA

  1. Log in to the NRAD console.
  2. Click the User Profile icon in the top-right corner.
  3. Select Manage to open the MFA page.
  4. Follow the prompts to set up multi-factor authentication for your account.

NOTE: When disabling MFA, a confirmation message will appear, and you will have the option to enable it again.
