Hotfix: SA_8.1.007

Summary: Test Differential Scan File Skipping - There is a bug with SQLite where it incorrectly read datetimes stored in UTC as being in local. The impact of this would be if the local time was ahead/behind of when the file was modified (in UTC) it would either skip the file (as it thought it hadn't been modified even though it had) or it would scan the file (thinking it had been modified when it hadn't). This fix addresses that issue and adds logging messages as to the reasons files are being skipped - file hasn't been modified, unsupported file type, too large (in cases of sdd scanning where the file was larger than the max threshold set during scan config).
