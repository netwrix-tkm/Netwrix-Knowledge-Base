Summary: SI-don't alert on inherited permission changes

Issue: Summary: SI-don't alert on inherited permission changes
Behavior before Fix:
SI reports permission changes for parent and all child objects
Behavior after Fix:
Now it can set FQDN or NetBIOS name of the host depending from the 'File System StealthAUDIT' Policy settings. This setting can be configured on the 'File System Agents' tab.
Changes since Last Hotfix:
SI Console: added new option in the Configuration/File Monitor Settings: 'FS inherited permissions filtering'
This option will change the 'DISCARD_INHERITED_ACL' key in .INI files.
�Enable 'FS inherited permissions filtering' - SI will report only initial permission change and filter out inherited changes.
