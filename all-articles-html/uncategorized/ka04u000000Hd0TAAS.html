Hotfix: SD_2.0_004

Summary: This Hotfix addresses a variety of issues relating but not limited to: CPU Usage by the Event Listener, Job Service issues with HTTPS configured, DCSYNC false positive detections, Email Service issues, Nested Group Member Login, and FIPS support.

Issue: Summary:
This Hotfix addresses a variety of issues relating but not limited to: CPU Usage by the Event Listener, Job Service issues with HTTPS configured, DCSYNC false positive detections, Email Service issues, Nested Group Member Login, and FIPS support.
Behavior before Fix:

  • Event Listener Service could over-use CPU when scanned by security/port scanning products
  • DCSync threat could still trigger for Domain Controller servers in some rare instances
  • Job Service would be unable to launch some jobs when HTTPS is configured
  • Email Service could send Blank Emails
  • StealthDEFEND could not be run in a FIPS Compliant operating system environment.
  • Members of many nested groups scoped with access for DEFEND would be unable to login to the console.
Behavior after Fix:
  • Event Listener properly handles invalid connections.
  • DCSync threat detection improved to prevent false positives
  • Job Service now properly handles HTTPS in all configurations.
  • Email Service has adding protection/handling to not send invalid emails.
  • Optional FIPS complaint configuration option now exists for StealthDEFEND.
  • New Group Evaluation method for Nested Group Members scoped with Access for StealthDEFEND.
