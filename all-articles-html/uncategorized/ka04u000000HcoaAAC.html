Hotfix: SA_7.2_024

Summary: This is a fix for the service account being set as a site collection administrator for a user private site when scanning sharepoint online. New functionality will allow administrators to scan private sites with the service account and STEALTHAudit will remove the permissions after the scan is complete. This fix adds ForceSiteAdminForInaccessiblePrivateSites configuration setting to the SBSPOSvr.exe.config file. When the value is set to false (default), scanning private sites that the service account does not have specific access to will generate an access denied error and the private site will be skipped. When the value is set to true the service account will be used to scan the private site and then the permissions will be removed from the site collection after the scan is completed.
