How to Install the signed Threat Prevention agent on a host without an internet connection

Summary: A customer would like to utilize the Threat Prevention signed agent, however the domain controller does not have access to the internet.

Issue: A customer would like to utilize the Threat Prevention signed agent, however the domain controller does not have access to the internet.

Instructions:

  • To bypass the certificate revocation check due to lack of internet on the Domain controllers do the following before installing the Threat Prevention signed agent:
    1. Install the Symantec certificate (Symantec Class 3 SHA256 Code Signing CA) to the trusted root of the certificate store.
    2. Locateï¿½ HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control
    3. Locate the ServicesPipeTimeout entry. (If the ServicesPipeTimeout entry does not exist, you must create it.)
    4. Change the value to 60000 (decimal), and then click OK.
    5. Restart the computer for the change to take place.
