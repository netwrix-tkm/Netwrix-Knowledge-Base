Hotfix: SDD_8.0_004

Summary: Allows the file scan timeout parameter to be configurable by adding xml tag in Job xml. Value is set in minutes. To make it so a file will never timeout, input a really high value for the timeout (Ex: 9999999).

Instructions:

  1. Uninstall previous version of SDD Add-on from local SA Console (if applicable).
  2. Install SensitiveDataAddon.msi on your local SA Console.
  3. Open the job xml of the SDD scan job. Under xml tags, add tag to configure the file scan time out value in minutes. It should look like this:


The above screenshot has the timeout value set to 5 minutes.

4. Launch StealthAUDIT. Run the 'Update Proxy Service' query under the Maintenance tab in the FSAA data collector against your proxy servers (if running proxy service).
