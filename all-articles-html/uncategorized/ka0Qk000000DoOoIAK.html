Hotfix: SA_10.0_001

Summary:

Summary: Web Server: Favorites report shows wrong generated time stamp on it

Issue/Request: When upgrading Access Analyzer from 8.2 to 10.0, a 'Favorites' report that is regenerated in 10.0 still shows the old time stamp generated in 8.2.

Fix/Enhancement: Web Server now gets the report path from PublishedReports.xml, where it is updated, instead of searching on file system, where the report resides in a legacy folder (v2). The time stamp shown will be the most recent one.
