How To: Change the Log Events from Error to Debug

Applies To:

Netwrix Directory Manager 11 � Portal

Business Scenario:

In Directory Manager, logs help administrators track system activities, diagnose issues, and troubleshoot errors. By default, the logging level is set to Error, meaning only critical issues are recorded. However, in certain situations�such as investigating unexpected behavior, debugging a failed process, or working with support�you may need more detailed logs.

Changing the log level from Error to Debug allows Directory Manager to capture additional details, providing deeper insights into system operations. This helps in identifying root causes, analyzing system behavior, and resolving issues more effectively.

In this article, we will walk through the steps to modify the log settings and set the logging level to Debug.

Solution:

To enable detailed logging, you need to update the log settings for each application individually.

Steps to Change Log Events from Error to Debug:

  1. Go to the Admin Center.

  2. Navigate to Applications.

  3. Open the Settings of the application where you want to modify the log settings.

  4. Go to Deployments.

  5. Select Logging.

  6. Change File Logging and Windows Logging from Error to Debug by selecting the option from the dropdown list.

  1. Click Save to apply the changes.

  2. Repeat the same process for the following applications:

    • Replication Service

    • Admin Center

    • Data Service

    • Security Service

    • Email Service

    • Mobile Service

    • Scheduler Service

After completing these steps, Directory Manager will capture detailed logs for better system monitoring and troubleshooting.
