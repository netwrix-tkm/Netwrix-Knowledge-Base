GSM Self Check: �Check Available Memory�

Cause:

  • There is less than the recommended amount of memory installed on your GSM VM.

Resolution:

  • Ensure that the GSM VM is running on a supported Greenbone Hypervisor (LINK) using the Greenbone Support Package

\support-package\gsm-support\kernel-journal

Line: kernel[]: info: Hypervisor detected: --HYPERVISOR--

  • Check how much memory is assigned to the GSM VM using the Greenbone Support Package

\support-package\gsm-system\commands.txt

Line: Executed command: 'vmstat -s'.

Output: 

7137672 K total memory
685820 K used memory
3014812 K active memory
3009136 K inactive memory
722312 K free memory
228416 K buffer memory
5501124 K swap cache
0 K total swap
0 K used swap
0 K free swap�

  • Ensure that the amount of resources that are assigned to the VM hosting the GSM, are in line with the specific GSM System Requirements (LINK).

    • If not enough resources are assigned to the VM, add accordingly.

    • If the total memory reading does not match with the memory assigned to the GSM VM through the hypervisor, this is usually due to address space being taken by PCI devices and the details differ from hypervisor to hypervisor. In this scenario, raising the memory assigned to the GSM VM may resolve the issue.
