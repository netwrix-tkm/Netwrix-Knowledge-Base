Lost Job Settings After Upgrading to 10.0

Summary:

An issue has been identified in Change Tracking that causes conflicts when upgrading to SA 10.0. This issue was due to outdated xml files in the out of the box solutions that have recently been updated with 10.0.
We have addressed this fix for future upgrades.

However, in order to limit any issues that may arise with any customer that plans to upgrade in the future, a tool has been created to fix change files to be compatible for upgrades.

�

We recommend current customers on 9.0 and 10.0 that have not yet experienced this issue to run this tool prior to any future upgrade.

Customers that have already upgraded to 10.0 and experienced this issue should run this tool prior to any future upgrade.

�

Submitted by:

Michael Burrofato

Affected Versions:

9.0+

Affected Module:

FixChanges.exe

Comments:

This tool has a pre-requisite:
If using SA 9.0, please ensure you have applied Hotfix SA_9.0_088 or are using CU version 9.0.0.153 or higher.

If using SA 10.0, please ensure you have applied Hotfix SA_10.0_019 or are using CU version 10.0.0.159 or higher.

�

Hotfix Instructions:

  1. Unblock the zip file in the Windows property dialog, if an unblock button exists there.
  2. Close all instances of StealthAUDIT (check task manager under processes for all users)
  3. Run FixChanges.exe
  4. Restart StealthAUDIT
