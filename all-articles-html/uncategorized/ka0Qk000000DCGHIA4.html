Walkthrough: Configure Notification

Email notifications are used by GroupID to notify users, group owners and managers when changes are made to directory objects, such as when a group�s membership is updated or when a user�s profile is modified.

For notifications to be generated and sent to the right audience, certain configurations must be in place. In this article, we provide a walkthrough on how to configure notifications for an identity store.

To configure notifications:

  1. In GroupID Management Console, click the�Identity�Stores�node.
  2. On the Identity Stores tab, double-click the required identity store to open its properties.
  3. Select the�Configurations�tab and then click�Notification�in the left pane.

SMTP server:

Provide the address of the SMTP server that will be responsible for routing the�notifications for the identity store. You can provide the IP address or FQDN of the server, as shown in the image above.

From email address:

This is the address from which the notifications will be sourced or the address that will appear as the source of the notifications in the recipient's inbox, as shown below. Some organizations prefer an address like�no-reply@domain.com; however, in this example, we have configured it as�notification@demo.com.

Port:

The port number for communicating with the SMTP server.

Test:

After providing an email address and port number, click�Test�to test the server settings. It will ask for a destination address to send the notification to. If everything goes well, you will get a success message and a test notification will be sent to the destination address.

Use SMTP User Authentication:

By default, the credentials of the logged-in user are used to connect to the SMTP server. However, if this user account does not have rights on the server or if you want to use a different account for connecting, select the�Use SMTP User Authentication�check box.

Upon selection, the�Username�and�Password�boxes become available, where you can provide credentials of an authorized user account on the SMTP server.

SSL Enabled:

Select this check box if the SMTP server is SSL (Secured Socket Layer) enabled.

Recipients - To and CC:

Notifications that are generated by GroupID for any event will be sent to recipients whose email addresses you provide in the�To�and�CC�boxes.

    • Notify logged-in users about changes being made to objects
      Select this check box to send email notifications to the logged-in user for changes they make to directory objects using GroupID.
      This setting applies only to mail-enabled users.
    • Notify owner/manager
      Select this check box to send email notifications to the primary owner and additional owners (for groups), and manager (for user/contact) about changes made to the respective directory objects.
    • Notify object being modified
      Select this check box to send email notifications to objects (group, user, contact) when they are modified.
      For a group, group members are notified.
      For contact and user, the particular contact or user is notified about the changes.
    • Notify public group owner of membership change
      Select this check box to send email notifications to the primary and additional owners of a public group on membership change.
      A public group is open to all users; anyone can join it or leave it without requiring any permissions. Selecting this check box notifies group owners by email whenever someone joins or leaves the group.
    • Notify newly added group members on addition
      Select this check box to send email notifications to objects when they are added to the membership of a group.
    • Notify removed group member on removal
      Select this check box to send email notifications to objects when they are removed from the membership of a group.

Password Portal URL:

Select the URL of a Password Center portal. This URL will be included in password expiry warning emails to redirect users to the Password Center portal.

Membership Lifecycle Notification Options:

Membership life cycle notifications are sent to users when the Membership Life Cycle job adds or removes them temporarily from a semi-private or public group�s membership.

In the Membership Lifecycle Notification Options area, select your desired options.

    • User joins the group
      Sends email notifications to users when they temporarily join a group.
    • User leaves the group
      Sends email notifications to users when they temporarily leave a group.
    • X days before user leaves the group
      Select this check box and specify a certain number of days, say 2. The user receives a notification 2 days before he or she is removed from group membership.

Managed By Notification Options

You can set up email notifications for objects when the Managed By life cycle job adds or removes them as temporary additional owners for groups and temporary additional managers for users.

When a group is set or removed as a temporary additional owner, all its members are notified.

In the Managed By Notification Options area, select your desired options.

    • Make options same as membership lifecycle
      Applies the membership lifecycle notification settings to the managed by life cycle and disables the settings in the Managed By Notification Options area.
      For example, if User joins the group is selected for Membership life cycle, it would have the behavior as discussed for Object is added as an additional owner/manager.
    • Object is added as additional owner/manager
      Sends an email notification to an object when the Managed By life cycle job adds it as the temporary additional owner of a group or the temporary additional manager of a user. The primary owner or primary manager is also notified respectively.
    • Object is removed as additional owner/manager
      Sends an email notification to an object when the Managed By life cycle job removes it as the temporary additional owner of a group or the temporary additional manager of a user. The primary owner or primary manager is also notified respectively.
    • X Days before object is removed as additional owner/manager
      Select this check box and specify a certain number of days, say 2. The temporary additional manager/owner receives a notification 2 days before he or she is removed as additional manager/owner.
  1. Click Apply and then OK.
