Troubleshoot: Unable to Sort Groups by DisplayName.

Applies To:

Directory Manager 11 �

Business Scenario:

The Directory Manager portal enables us to sort group listings by different attributes (such as display name, description, and owner) in both ascending and descending order.

In the Directory Manager portal, go to a group listing and simply click in a column header to sort the groups by that attribute. Then use the arrow heads to arrange the listing in ascending or descending order.

Here, groups are sorted by display name in ascending order, as indicated by the upward arrow in the�Display Name�column header.

Problem Statement:

The groups in the My Groups listing do not get sorted in ascending order (alphabetically) when I try to sort them by the DisplayName attribute. The screen displays a �Loading� message, but the system fails to sort the listing.

More Information:

The display name is not a mandatory attribute for groups when you create them in Active Directory. As a result, these groups do not have a display name. (You can verify this by checking the DisplayName attribute in the attribute editor of a group in Active Directory.)

On the other hand, the display name is a mandatory attribute when you create groups using Directory Manager. This ensures that the DisplayName attribute is never left empty and that every group has a display name.

Issue Resolution:

When you sort the group listing by the DisplayName attribute, the Directory Manager portal treats the display name as a null value for groups that do not have one. So, it uses the�common name�of those groups in place of display name, which results in the�faulty sorting issue.

Once the DisplayName attribute is populated for all groups, the portal will properly sort group listings in terms of display name.
