What to do When the Access Analyzer Job Tree Disappears

Summary: From time to time customers email stating that their job tree is no longer populated within Access Analyzer. This is typically caused by blank or corrupted job XML files.

Issue: When a customer reaches out stating that their job tree is missing from the console or only partially loading it is typically caused by a blank or corrupted Job XML file somewhere in the job tree. These blank or corrupted XML files are typically caused by a forced reboot of the server during a job execution, or when disk space for the drive hosting the Access Analyzer application is depleted.
Emails describing this issue will typically come in during Patch Cycles. The XML files are blank most times because during a scheduled task execution the XML file is held open until the job finishes at which time it is saved. If the server is forcibly rebooted then the XML doesn't get a chance to properly save.

Instructions: The easiest way to resolve this issue is to get on a call with the customer and find the job with the corrupted XML and restore it to the working state. Step to find the corrupted XML are as follows.

  1. Launch Access Analyzer and observe the state of the job tree
    1. Empty
    2. Partially displayed
  2. For Empty Job Tree take the following steps
    1. Right click the jobs folder at the top of the tree and select Refresh Tree
    2. Note the last job to appear in the tree
    3. Right click the parent group of the last job displayed in the tree and select explore folder
    4. Explore the first job not displayed in the tree and find the XML file. Typically the XML file will be 0kb in size
    5. Go into the output folder and copy the job XML file back into the main job folder overwriting the corrupted or blank XML
    6. Go back into Access Analyzer and refresh the Jobs Tree
    7. Repeat Until Entire Jobs Tree is visible.
  3. For Partially Displayed Job Tree
    1. Right click the parent group of the last displayed job in the tree and select the explore folder option
    2. Explore the folder of the first job that wasn't displayed
    3. Locate the job XML file
    4. Navigate to Job Output folder and copy the job XML file back into the primary job folder
    5. Go back to Access Analyzer and refresh tree
    6. Repeat until entire jobs tree is visible
