Hotfix: SAM_4.1_016

Summary: 1. Defend and SAM only showing events for 1 of 6 DCs 2. Logging successful folder list actions for VNX devices 3. C-mode NetApp: Monitor NFSv4 by default 4. NetApp C-mode: Office filtering must not affect NFS activity 5. NetApp: No NFS Read events when a file is copied outside of the volume

Issue: Summary:

  1. Defend and SAM only showing events for 1 of 6 DCs
  2. Logging successful folder list actions for VNX devices
  3. C-mode NetApp: Monitor NFSv4 by default
  4. NetApp C-mode: Office filtering must not affect NFS activity
  5. NetApp: No NFS Read events when a file is copied outside of the volume
Behavior Before Fix:
  1. Under certain conditions SAM would not upgrade AD Module.ï¿½ Causing loss of AD activity.
  2. Folder list actions / Folder Read events were not implemented for EMC Devices.
  3. SAM didn't monitor NFSv4 by default for CMode Netapp.
  4. Office filtering was effecting CMode Netapp NFS Activity, causing loss or delay of activity.
  5. NFS Read events were not logged copying to local filesystem.
Behavior After Fix:
  1. Implemented fix that would upgrade AD Module correctly.
  2. Implemented Folder list actions / Folder Read events for EMC devices.
  3. SAM now monitors NFSv4 events by default for CMode Netapp.
  4. Now office filtering does not effect NFS Activtiy.
  5. We now include 'close' in fpolicy nfsv4 events to report reads more accurately.
