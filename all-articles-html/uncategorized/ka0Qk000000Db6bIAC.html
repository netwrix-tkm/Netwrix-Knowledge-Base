Threat Prevention Basic Troubleshooting

Summary: basic steps to troubleshoot and verify Threat Prevention functionality

Issue: To verify basic Threat Prevention functionality in the Threat Prevention Windows 32bit console:
�

Instructions: �

  • license manager
    • what modules are licensed
      • unlicensed modules events will be discarded
    • when does the license expire
      • expired license will trigger Windows agents to store events locally until license is not expired or local Windows agent queue overflows
  • agents frame
    • events in last sixty seconds
      • if a Windows agent or set of Windows agents consistently over a consecutive timeframe records thousands of events then review the latest events and enabled policies
    • events in queue
      • if a Windows agent or set of Windows agents consistently records millions of events in queue then review the enabled policies, network bandwidth, and database resources
    • Windows agent version
      • always useful to be on the latest Windows agent except if the Windows agent is on an unsupported Windows agents
    • AD Directory event latency
  • alerts frame
    • queue overflows
      • review the enabled policies, network bandwidth, outbound queue folder, Windows agent logs, and available disk space
    • lost connections
      • review network bandwidth, ports 3136, and 3739
  • investigation
    • latest thousand events
      • review policy, or policies with most event, and event times
    • latest one hundred thousand events
      • review policy, or polices with most events, and event times
  • logging
    • installation logs
      • :\users\\apdata\local\temp\ or :\users\\apdata\local\temp\\
    • debug logs capture the most detail
      • Threat Prevention Windows agents logs
      • Threat Prevention Enterprise Manager logs
      • Threat Prevention Scheduler service logs
      • SI Events log
        • Group Policy Object monitoring and lockdown events are in the SI Events logs
    • file system monitoring events in '*.tsv' log files
    • operating system logs
      • service or services stopping without notice
      • .NET errors
