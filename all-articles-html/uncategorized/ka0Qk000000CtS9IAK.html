Issue:

e.g. left one scan running overnight and it still hadn�t got past that step the next morning.
After enabling logging for each scan, all scans are getting stuck at �Gathering gpo data�.

Occurs in versions 3.2.0.1 and 3.3.0.1

Fix:
Check the correct ports are open:

  • LDAP (tcp/389): Used for querying the Active Directory.
  • ADWS (tcp/9389): Used for Active Directory Web Services.
  • SMB (tcp/445): Used for file sharing and other network services.
  • DNS (tcp/53): Used for domain name resolution.
  • RPC (tcp/135): Used for remote procedure calls.

Check trace logs and find errors/delays scanning MSI files.
Disabling some DFS paths "correctly" can fix this issue
