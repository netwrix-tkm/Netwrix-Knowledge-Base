Behavior before Fix:
  1. ï¿½
  • Windows 2k19:
Kerberos events not captured due to signature changes in 'KdcVerifyKdcRequest', 'HandleTGSRequest', 'I_GetASTicket' & 'KdcGetTicket'
  • Windows 2k16
Kerberos events not captured due to signature change in 'HandleTGSRequest'
  1. SI Agent does not inject into Exchange process because of new versions of .Net libraries.
  2. SI Agent does not inject into Exchange process because of new versions exrpc32.dll
  3. LDAP search events not captured due to signature changes in 'DirSearchNative'
  4. When gpLink attribute in AD is modified then SI Agent parses this value and presents it if the format "[DN;N;GPOName]" where:
"DN" is Distinguished Name of corresponding groupPolicyContainer object. For example: cn={31B2F340-016D-11D2-945F-00C04FB984F9},cn=policies,cn=system,DC=domain,DC=com,
"N" is number representing "Enforced" and "Link Enabled" flags state.
GPOName is the Display Name of this groupPolicyContainer object.
Previously code which was used to query Display Name was using incorrect Search Scope (Base) for corresponding LDAP query which caused in showing GPOName always as "[NOT FOUND]".
Behavior after Fix:
  1. Signatures adjusted on both platforms; Kerberos & LDAP events are captured normally.
Note: Patch also eliminates noisy 'pDirRead failed:' message in AD Monitor logs.
  1. Extend sig file to support new versions of .Net libraries
  2. Extend sig file to support new versions of exrpc32.dll
  3. Signatures adjusted for 2019 Server; LDAP events are captured normally.
  4. ï¿½Code is changed to use SubTree search scope.
