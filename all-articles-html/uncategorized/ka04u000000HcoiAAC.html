Mismatched GatesProxyIDs when scoping is different between FSAA and SDD scans.

Summary: A bug was found that would result in GatesProxyIDs being off when scoping is different for FSAA and SDD scans.

Issue: A bug was found that would result in GatesProxyIDs being off when scoping is different for FSAA and SDD scans.
Example Use case:
1. Create two shares "TestShare" and "TestShare2"
2. Scan and Bulk Import FSAA, note the GatesProxyIDs
3. Exclude "TestShare" for SEEK scan (Assuming its resourceID is lower than TestShare2) and Bulk Import
4. Look at ResourceMap - note that the GatesProxyID aligns to the DLPID and not the Mapped ID
