Summary: SI console warning "Operation Failed: Get AD Subordinates List" Unable to search, "operation failed: get ad subordinates list"

Issue: Summary:
SI console warning "Operation Failed: Get AD Subordinates List"
Unable to search, "operation failed: get ad subordinates list"
Behavior before Fix:
Process of searching AD objects is the WinConsole Select Objects dialog uses multiple threads. This is needed because if operation takes more that 1 second then UI shows progress dialog with possibility to cancel this operation. Previously the sequence of actions how two threads are communicating allowed getting of inconsistent values showing the wrong state of the search thread which can cause in different errors and exceptions.
Behavior after Fix:
Now logic in this place changes to set all values before the calling thread will get signal about child thread completion.
