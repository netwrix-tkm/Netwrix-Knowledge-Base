Hotfix: SA_7.2_SP1_064

Summary: Running the Mailbox Permissions Collection job, the memory consumption of Stealthbits.PowerShell.Server.exe process taking all available memory and never returning data in a collection that is not scoped. This happens in both scheduled and interactive executions. During the interactive execution the job status message stays hung on initializing and the log never gets past the PowerShell script that the data collector is attempting to execute.

- PrivateAssemblies\Stealthbits.StealthAUDIT.DataCollectors.ExchangePS.dll v. 7.2.109.0
- PrivateAssemblies\Stealthbits.PowerShell.Server.exe v. 7.2.40.0
