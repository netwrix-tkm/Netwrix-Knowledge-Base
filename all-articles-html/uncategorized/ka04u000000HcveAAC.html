Summary: Agent Disconnection Issues

Issue: Summary: Agent Disconnection Issues
Behavior before Fix:
SI Agents may stuck in 'Lost Connection' state on environment with slow network and default grpc settings
Old grpc settings for EM:
grpc.keepalive_time_ms = 1000
grpc.keepalive_timeout_ms = 2000
grpc.keepalive_permit_without_calls = 1
grpc.http2.min_time_between_pings_ms = 3000
grpc.http2.min_ping_interval_without_data_ms = 3000
grpc.http2.max_pings_without_data = 0
grpc.enable_deadline_checking = 1
grpc.max_receive_message_length = -1
grpc.max_send_message_length = -1

Old grpc settings fro Agent & WinConsole:
grpc.keepalive_time_ms = 1000
grpc.keepalive_timeout_ms = 2000
grpc.keepalive_permit_without_calls = 1
grpc.http2.min_time_between_pings_ms = 5000
grpc.http2.min_ping_interval_without_data_ms = 5000
grpc.http2.max_pings_without_data = 0
grpc.max_receive_message_length = -1
grpc.max_send_message_length = -1
Behavior after Fix:
This is potential fix. It includes grpc settings adjustments and exposing grpc settings to config files.
Now its possible to specify any grpc setting for EM, Agent & WinConsole via config file like this:

Default grpc settings were changed.
EM:

grpc.keepalive_time_ms = 2000
grpc.keepalive_timeout_ms = 10000
grpc.keepalive_permit_without_calls = 1
grpc.http2.max_pings_without_data = 0
grpc.enable_deadline_checking = 1
grpc.max_receive_message_length = -1
grpc.max_send_message_length = -1
grpc.http2.min_time_between_pings_ms = 2000
grpc.http2.min_ping_interval_without_data_ms = 10000

Agent & WinConsole:

grpc.keepalive_time_ms = 2000
grpc.keepalive_timeout_ms = 10000
grpc.keepalive_permit_without_calls = 1
grpc.http2.max_pings_without_data = 0
