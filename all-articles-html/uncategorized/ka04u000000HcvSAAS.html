Issue: Summary:
Lockdown policy does not block accounts with $ in samaccountname
Behavior before Fix:
Previously blocking rules building engine accepted as possible values for perpetrator filter only groups, users, and well-known SIDs.
Behavior after Fix:
Now logic is updated to accept also 'other' objects if it has SID.
