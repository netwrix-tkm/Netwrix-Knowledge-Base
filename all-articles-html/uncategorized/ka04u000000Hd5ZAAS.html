Issue: Summary:
Summary: This hotfix corrects an issue where an impersonation error appeared in the application log when locking or unlocking a Job.
Issue/Request: When a regular domain user (not an admin) with the Job Approver role in StealthAUDIT locked or unlocked a Job, an impersonation error appeared in the application log.
Fix/Enhancement: No impersonation error appears in the application log, and no system warning/error is shown.
