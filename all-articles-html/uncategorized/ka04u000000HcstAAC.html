Unprintable data in LDAP events

Charles Schwab wants to set up a monitoring policy for specified service accounts. Observed after setting up a monitoring policy that group changes were not captured by the policy.
After fix:
SI-3423
Fix details:
Previously there was already implemented the logic how to better convert 'DomainGuid', 'schemaIdGuid', and 'NtVer' attributes in LDAP filter to string value. Now this list extended and includes also 'objectGuid' and 'msExchMailboxGuid' attributes.
SI-3427
Fix Details:
In addition to existing logic of evaluating filter conditions for AD event in AD Changes event type added following rules:
1. If 'AD Objects' filter didn't match using old logic but event has 'member' attribute then in addition to the current logic 'AD Object' filter will also be evaluated for each value in the member attribute.
2. If 'AD Context' filter didn't match using old logic but event has 'member' attribute then in addition to the current logic 'AD Context' filter will also be evaluated for each value in the member attribute.
Logic of other filters wasn't modified and will work as before.
