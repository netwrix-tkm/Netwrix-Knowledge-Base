How to: view list of Objects Owned by a User in GroupID Portal

Applies To:

GroupID 11 ï¿½ GroupID Portal

Problem Statement:

We have users who are owners of numerous groups in our domain. We want the other users/administrators to be able to view the list of all the groups that a specific user is an owner of. Is there a way to view/show all the groups owned by a user in the groupID portal?

Methodology:

Note: Before making any changes to the current environment, it is highly recommended to always create a backup/snapshot/checkpoint of the groupID server.

GroupID enables us to customize the GroupID portal based on our requirements. We can add a wide range of functionalities to the portal using the GroupID Admin Center.

In this article, we will be making changes to the design of the portal to add a new field to the properties of the Users that will enable us to view the list of groups owned by users. We will be using the managedObjects attribute to map the newly created field. By default, this attribute is only replicated for Groups in groupID. To use managedObjects for the Users, we need to include them in the replication. The steps guide you through the whole process.

The default properties page of Users looks as shown below:

Steps:

  1. In the GroupID Admin Center, go to the Applications tab in the left navigation bar.
  2. Click on the Ellipses button for the portal you wish to make changes to, and then click on Settings.

  1. Once the portal settings open, click on the Identity Store name under Design Settings for which you want to make the design changes.

  1. Click on the Properties button and then select User from the Select Directory Object dropdown list.
  2. Click on the Plus to create a new Category on the User Properties.

  1. Provide a Name to the new Category and click on the +Add Field button.

  1. Select managedObjects from the Field dropdown list, provide a Display Name to the new field, and then select Display Type as MultiValue.

  1. Click OK and then click Save to save the changes.
  2. Refresh/relaunch the GroupID Portal and verify by going into the properties of any User. There will be a new tab for Owned Objects available there as shown by the picture below where all the groups owned by the user are listed in the Owned Objects tab within User Properties.
