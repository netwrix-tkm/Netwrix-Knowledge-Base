SIWindowsAgent_4.0.0.682

Summary: Group policy object monitoring not monitoring

Issue: Before fix:�
If user selects some particular GPO for monitoring and has several DCs in the environment which use different location for SYSVOL folder (for example, on DC1 it is "C:\Windows\SYSVOL" and on DC2 it is "E:\Windows\SYSVOL") then events will not appear for some of these DCs. This is because SI Policy filter used full path to GPO files to compare if this event must be ignored or not.
After fix:
Instead of comparing file path, now it will compare only GUID part of this path.
