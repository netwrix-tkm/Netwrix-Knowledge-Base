Instructions: 1. Exit Access Analyzer console
2. Update all DC files in your StealthAUDIT\DC directory
3. Update all PrivateAssemblies files in your StealthAUDIT\PrivateAssemblies director
4. Stop Agent service on SharePoint Server
6. If updating an agent on SharePoint 2010: update the files in Agent installation directory with the DLLs in the NET35 Folder
If updating an agent on SharePoint 2013: update the files in Agent installation directory with the DLLs in the NET40 Folder
7. Restart the Agent service on the SharePoint Server
8. *Note:* this fix was to surface previously hardcoded settings to allow end users to configure them. One of the settings: MaxCutOffFilesForPattern is now 0 (this setting was previously

set to 100, essentially telling the DLP add on to stop searching for that pattern after finding 100 unique instances of it in the SharePoint Environment). The option can now be found in the

job xml (search MaxCutOffFilesForPattern) if someone wants to change it back.
