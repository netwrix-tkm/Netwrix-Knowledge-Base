Netwrix Change Tracker Removing Redis Dependency

This document provides the steps required to remove Change Trackerï¿½s dependency on the Redis service in order to prevent the need to configure a remote Linux server running Redis.

This configuration is the result of detected vulnerabilities with the shipped version of Redis within Netwrix Change Tracker and the inability of Netwrix to upgrade the Redis version to a later Windows version.

The preferred option for addressing Redis vulnerabilities is to configure a standalone Linux server, install Redis and alter the Change Tracker configuration to use the remote Redis server. Steps to configure the remote Redis server can be found in the Decouple the Redis Application document.

The removal of the need for the Redis server will also prevent Change Tracker from using the worker services and so will return the 7.3 installations to 7.2 levels of processing and performance. This configuration is only recommended for customers with under a 1000 license.

Steps to Disable Redis

All configurations for this document were carried out using Netwrix Change Tracker 7.3.0.1 the GA released at the time of testing.

1 - Stop the IIS service, both worker services, and the Redis service

2 - Configure the Netwrix Hub Workers and Redis service to disabled

3 - Disable the distributed cache by editing the appsettings.json in C:\inetpub\wwwroot\Change Tracker Generation 7 (NetCore) Hub\Configs

4 - Edit the file Web.config in C:\inetpub\wwwroot\Change Tracker Generation 7 (NetCore) Hub to include the environment variable NNT_WORKER_ROLE = AllPurpose

5 - If you prefer, rather than disabling the Redis service, move to control panel and uninstall the application.

6 - Restart IIS
