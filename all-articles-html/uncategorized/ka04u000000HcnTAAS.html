Do Sensitive Data Discovery Add-On (SDD) binaries get updated with the FSAA FS_UpgradeProxy Job?

Summary: After an upgrade of the StealthAUDIT console, does the Sensitive Data Discovery Add-On get updated after you run the FS_UpgradeProxy Job on the proxies?

Issue/Question: In your environment you are utilizing proxies to offload file system scanning capability from the StealthAUDIT Console (which is the default setting) to remote windows servers. You have just upgraded your StealthAUDIT Console and also installed the newest version of the Sensitive Data Discovery Add-On and now you want to upgrade all your remote Windows proxies that have the FSAA Proxy application installed. You are not sure if the FS_UpgradeProxy Job found in instant jobs library will update the SDD binaries, or if the Sensitive Data Discovery Add-On will need to be reinstalled on the proxies manually.

Answer:ï¿½ Yes, SDD binaries do get updated with the FS_UpgradeProxy Job, so there is no need to manually uninstall the Sensitive Data Discovery Add-On on each individual proxy and reinstall the newer version. When you update the binaries this way, the version of the software will not show as changed in programs and features, so do not make the mistake of looking here to see if the product is on the newer version. You will have to check the modified date of the files in the %sainstalldir%\fsaa\ and %sainstalldir%\fsaa\sensitive data\ to see if they match the time of when the FS_UpgradeProxy Job was run.
