Title:
NAM_7.0_004
Summary:
  1. Previously provided Home Directory fix also does not work
  2. Upgraded to 7.0 from 6 and activity agents on windows server started spitting back occasional errors
Behavior Before Fix:
  1. Activity Monitor didn't detect custom EPEL correctly, connection test would report error, and agent install would report an error; Issue removing firewall port during linux agent uninstall.
  2. Activity Monitor reported a misleading error when no NFS shares existed on windows host.
Behavior After Fix:
  1. We now check for available lttng-ust library itself instead of epel; updated removal of firewall port during linux agent uninstall.
  2. Error has been updated to an info message that is logged when windows host doesn't have any nfs shares.
Changes Since Last Hotfix:
  • 302266: API: GetStatusAsync failed. Can't create client. The socket connection was aborted.
  • 301733: Checkbox in the Collect logs window doesn't work correctly
  • 302783: Linux Agent - Check for individual required libraries instead of EPEL
  • 303102: Linux Agent - If EPEL and liblttng-ust is not installed or avaialble, agent install error is empty
