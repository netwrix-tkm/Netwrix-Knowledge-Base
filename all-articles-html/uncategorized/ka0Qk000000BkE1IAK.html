SQL transaction issue due to which Netwrix Access Analyzer console is not accessible

Symptom Access Analyzer Console hangs at launch during "Test SQL Connection" Cause long running queries on the SQL server are blocking the test query from completing. In this instance the queries were related to the usage statistics feature that Access Analyzer reports back to Netwrix. Unchecking "Send Usage Statistics" did not prevent these queries from running. Resolution the *.proc files located at C:\Program Files (x86)\STEALTHbits\StealthAUDIT\SP\SQLScripts\UsageStatistics had to moved out of the folder. After this action the console completed the Test SQL Connection quickly. R&D is investigating why unchecking the "Send Usage Statistics" did not prevent they queries from running. ---------- The customer has disabled the "Usage Statistics" setting within Access Analyzer.
