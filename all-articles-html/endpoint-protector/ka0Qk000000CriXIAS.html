[UPDATE]- EPP UI slowness or other performance issues

Symptom
Slowness or performance issues with the Endpoint Protector (EPP) server UI.

Cause and Resolution

  1. Large Amount of Logs or File Shadows

    • Cause:�Accumulation of logs or file shadows on the EPP server.
    • Resolution:
      • Check Appliance > Server Information for the number of logs and file shadows.
      • Delete older or unnecessary logs, or archive them using Audit Log Backup functionality.
      • Externalize logs using External Storage or SIEM integration functionality.

  2. Backend Services Being Affected

    • Cause:�Backend services might be disrupted.
    • Resolution:
      • Reboot the EPP server from Appliance > Server Maintenance > Reboot or from your hosting platform (VM, AWS, Azure, GCP).

  3. Disk Space Issues

    • Cause:�Insufficient disk space.
    • Resolution:
      • Check Appliance > Server Information for disk space details.
      • Raise a Support ticket for investigation and removal of unnecessary files.
      • If hosted on your end (VM, AWS, Azure, GCP), allocate extra disk space and contact support for disk resizing.

  4. Resource Issues

    • Cause:�Insufficient CPU cores and RAM.
    • Resolution:
      • If hosted on your end (VM, AWS, Azure, GCP), add extra CPU cores and RAM.
      • Contact support via a Support ticket to allocate additional resources on the EPP server.

  5. EPP Server Upgrades or Audit Log Backups Running

    • Cause:�Performance can decrease during upgrades or backups.
    • Resolution:�Expect temporary UI performance decrease during and shortly after these processes.

  6. Running Older EPP Server and Client Versions

    • Cause:�Older versions may not perform optimally.
    • Resolution:�Always use the latest EPP server and client versions available as a best practice.
