Staging the Endpoint Protector Server

To start using Endpoint Protector, you must first make a server instance available. The server is where you will configure all endpoint controls and behaviors, and it serves as the platform for deploying the Endpoint Protector agent to endpoint systems.

�

There are two main options for server management: Customer-Managed and Provider-Managed.

If you choose the Customer-Managed option, the server can be installed either on-premises or in a hosted cloud environment.

�

The On-Premises option allows you to set up a virtualized image within your organization's local area network (LAN). Supported virtualization platforms include, but are not limited to, VMware and Hyper-V.


The Hosted Cloud deployment method enables you to use your own Amazon Web Services (AWS), Azure, or Google Cloud Platform (GCP) instance.


For more detailed information about each of these deployment options, please refer to the resources section on our website:

https://helpcenter.netwrix.com/category/endpointprotector

�

Here, you will find manuals for "Virtual and Hardware Appliance Deployment" and "Cloud Services Deployment."

�

Alternatively, if you require a Provider-Managed setup, an instance of Endpoint Protector can be provisioned in an isolated cloud environment. For more details about the Provider-Managed option, please contact your Netwrix Account Manager.

�

Please note: To use the Endpoint Protector Server in a production environment, you must have a License Key. After purchasing Endpoint Protector with the necessary module(s), your Account Manager will assign a license that can be installed within the Endpoint Protector Management Console (the configuration interface available on the Endpoint Protector Server). Instructions for accessing and configuring the Management Console are also available in the manuals referenced above.
