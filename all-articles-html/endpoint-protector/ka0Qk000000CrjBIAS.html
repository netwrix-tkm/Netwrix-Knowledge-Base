Capturing EPP client logs on a Linux machine

Question:

How to enable and capture Endpoint Protector Client logs on Linux machine?

Answer:

To capture regular epp client logs, please run the below commands:

  1. $ sudo su
  2. $ cd /var/log/epp-client/
  3. $ touch /var/log/epp-client/epp_client_daemon.log
  4. $ touch /var/log/epp-client/eppsslsplit.log
  5. $ sudo service epp-client-daemon restart
  6. Replicate the issue.
  7. $ cp /var/log/epp-client/epp_client_daemon.log /home/user/Desktop/
  8. $ cp /var/log/epp-client/eppsslsplit.log /home/user/Desktop/

Submit Log files to Endpoint Protector Support Team.

A restart of a system clears existing log entries and sometimes it is necessary for the EPP client to retain log entries after a system restart.

To continue capturing further logs after the system has been restarted the EPP client would need to capture logs in append mode.

�

To capture epp client logs in append mode, please run the below commands:

    1. $ sudo su
    2. $ cd /var/log/epp-client/
    3. $ touch /var/log/epp-client/epp_client_daemon_append.log
    4. $ touch /var/log/epp-client/eppsslsplit.log
    5. $ sudo service epp-client-daemon restart
    6. Replicate the issue, or wait for the issue to occur.
    7. $ cp /var/log/epp-client/epp_client_daemon_append.log /home/user/Desktop/
    8. $ cp /var/log/epp-client/eppsslsplit.log /home/user/Desktop/

Submit Log files to Endpoint Protector Support Team.
