How to De-Obfuscate Sensitive Data

Overview

By default, all sensitive data is obfuscated according to the Data Obfuscation Rules:

  • The first 4 characters are displayed if the length of the threat is fewer than 12 characters.
  • The first 6 characters are displayed if the length of the threat is greater than 12 characters.

Specific use cases:

  • For credit cards, PCI Security Standards are implemented.
  • For SSNs, only the last 4 characters are displayed.
  • For Brazil ID (CPF), the first 3 and the last 2 characters are obfuscated.

Important: Data is not obfuscated for file-type threat, file-size threat, and date threat.

If you need to view sensitive data in a de-obfuscated form, you can do so by following the steps below. Note that there is no specific feature for this in the EPP console, but it can be achieved through configuration changes.

Instructions

  1. Log in to the EPP console.
  2. Go to Device Control > Global Settings > Debug Logging > Set Log Level.
  3. Select a log level: Error, Warning, Informational, or Debug.

    Note: Selecting Debug will generate a large number of log entries. If you only need de-obfuscated data, it is recommended to select Error.

  4. The Obfuscate Sensitive Data checkbox will appear ticked. Clear this checkbox so that EPP agents send sensitive data in a de-obfuscated form.
  5. Save the changes.
  6. Sensitive data will now be displayed in a de-obfuscated form.
