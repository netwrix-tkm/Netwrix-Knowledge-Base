[UPDATE]- How to deploy the EPP client using SCCM /msiexec

When trying to deploy the EPP client through SCCM please use the below format and syntax after you have downloaded the EPP client on your end:

msiexec.exe /i " msiPath " WSIP="EPP_server_IP" WSPORT="443" /qn /norestart

E.g.

msiexec.exe /i "C:\EPPClientSetup.msi" WSIP="192.168.7.108" WSPORT="443" /qn /norestart

Note1: /norestart is optional

Note2: if using FQDN please adapt the above template with the below:

msiexec.exe /i "EPPClientSetup.5.6.1.1_x86_64.msi" /qn /l*v "c:\temp\EPP_inst.log" WSIP=mycompany.endpoinprotector.server.com WSPORT=443

where EPPClientSetup.5.6.1.1_x86_64.msi is the name of the installer. If your installed also has an IP in the name, the IP needs to be removed from the MSI name.

where "mycompany.endpoinprotector.server.com" has to be replaced with the IP or the FQDN correspondent to your EPP server.
