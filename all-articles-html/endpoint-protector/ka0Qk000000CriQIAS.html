[UPDATE]- Alerts Not Sent' notification

If you receive an alert similar to the following :

"Please manually check Content Aware Report for any missed alerts between X and Y.

To avoid this situation, configure your Content Aware Alerts more granular (single users or computers; avoid using all users/groups/computers)."

This means there were to many email alerts generated and not all of them could be delivered yet.

We recommend that you review the configured email alerts from EPP server UI > Alerts > Device Control Alerts/ Content Aware Alerts, and possibly remove the ones which are not needed.

Please avoid targeting all users/groups/computers in the email alerts, and configure more granular, only target some of the machines (not all) if email alerts are needed.

Also please note that all logs/ events will still be available from server UI > Reports and Analysis section, if you disable email alerts.

It is recommended that you review logs under server UI > Reports and Analysis section, instead of using email alerts.
