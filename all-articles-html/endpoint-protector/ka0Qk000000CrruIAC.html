[UPDATE]- How to use Client Update Mechanism

Client Update Mechanism is giving the Administrator the possibility to update the Endpoint Protector client on 100 Computers per hour.

Note: The custom port you define in this section will be used when generating the client update download link instead of the default 443.


Steps to use Client Update Mechanism:

1. Go to System configuration -> System settings -> and check Enable Client Update V2.

2. Complete Use custom hostname field, if necessary: example.cososys.com


NOTE: If your Endpoint Protector Server has a hostname, fill this field with the hostname.
Example: epp.hosted.endpointprotector.com

NOTE: If you have an IP address instead of a hostname, leave this field empty


3. If the UI port is changed, complete Use custom port field with the desired port.

Example with hostname:
You have: epp.hosted.endpointprotector.com:8455
At Use custom hostname you complete with epp.hosted.endpointprotector.com and at Use custom port with 8455

Example with IP:
You access the appliance on 192.168.10.10:8000



Example with IP without port changed:
Check only Enable Client Update V2 and leave the other fields empty.
