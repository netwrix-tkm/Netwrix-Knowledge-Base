[UPDATE]- How to manually activate debugging logs on Windows

Overview
If you are experiencing problems with the Endpoint Protector Client, the UI Debug option , and the SupportTool are not an option, you need to activate debugging logs manually. Follow the instructions below.

Instructions

  1. Open the Command Prompt as Administrator.

  2. Navigate to the root of your C: drive.

  3. Create the log files:

    • eppclient.log
    • eppsslsplit.log

  4. Restart the Endpoint Protector process through Services.msc:

    • Ensure�Services.msc�is run as Administrator.

  5. Once the steps are completed, reproduce the issue while the debugging logs are active. Log information will be captured in these files.

  6. Share the logs with the support team for further investigation.
