[UPDATE]- Why is my smartphone detected by Endpoint Protector as multiple device type?

Depending on the smartphone manufacturer, different ways to display and manage connected devices are used (i.e. Sony shows only one device type while Samsung uses three different ones).
For devices that present themselves as differently to the operating system, the Endpoint Protector client will detect more devices (i.e. Windows Portable Device - MTP and Modem device - Internet sharing).
This is the desired behavior as it presents the advantage of a more granular control, based on the user???s actions (i.e. connecting the device as a mass storage device, enabling the USB de debugging, etc.)

Example for Samsung smartphones:

    • Device connected as storage device (MTP) is displayed as SAMSUNG Electronics Co., Ltd/ [phone model]
    • Device connected as MTP with the USB Debugging active is displayed as SAMSUNG Electronics Co., Ltd/ SAMSUNG Android ADB Interface
    • Device connected as a USB Modem is displayed as SAMSUNG Electronics Co., Ltd/ SAMSUNG Mobile USB Modem



    Example for HTC smartphones

      • Device connected as storage device (MTP) is displayed as High Tech Computer Corp. / HTC [phone model]
      • Device connected as MTP with the USB Debugging active is displayed as High Tech Computer Corp./ MTP USB Device
      • Device connected as a USB Modem is displayed as HTC/ ANDROID_PHONE
