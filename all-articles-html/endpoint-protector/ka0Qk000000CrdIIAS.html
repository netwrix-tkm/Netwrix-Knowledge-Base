[UPDATE]- Enable Encryption option is missing from the SIEM Integration configuration

When using the TCP protocol, you have the option to encrypt communication to each SIEM server. In order to do so, enable the Encryption setting and then Upload the root CA that was used to sign the server certificate for the SIEM server in .pem format.

If the

Enable Encryption option is missing from the SIEM Integration configuration that could mean that either the appliance is running on a older OS version (14.04) and does not posses the capability to use SIEM encryption or if the OS version is newer (18.04 or 22.04), to not have the syslog-ng service installed.

For the 1st scenario, if the OS is 14.04 or older, then a appliance with newer OS version has to be installed

For the 2nd scenario, for OS 18.04 and above, please contact support in order to check if the service is installed on the server.
