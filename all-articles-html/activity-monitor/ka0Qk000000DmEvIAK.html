Hotfix: SAM_6.0_018

Summary:
  1. Will doing an in-place upgrade of WS2012R2 to WS2019 have any adverse effects on SAM or SI
  2. Retention procedure not working correctly
  3. SI to SAM not posting to SD
  4. Netwrix Activity Monitor (SAM) agents crashing after upgrade to 6.0.865
Behavior Before Fix:
  1. After OS upgrade from 2012R2 to 2019 registry entry for INI file is removed.� As a results config can be lost.
  2. The retention procedure based its decisions on the content of the file, read from metadata.� This can cause issues and activity log can be accidentally removed.
  3. During upgrades SAM was not setting AMQP settings correctly. Also if master DC was set to have wrong AMQP settings, this would also be synced to set incorrect AMQP settings to all other DCs.
  4. In some cases Console and Agents would crash after upgrade.
Behavior After Fix:
  1. SAM now searches for existence of INI file and preserving config if registry entry is lost or removed.
  2. The retention procedure now uses the timestamp extracted from the file name to decide if the file needs to be removed
  3. SAM now sets correct AMQP settings during upgrade.
  4. Implemented potential fix.� Console and Agents should not crash after upgrade.
