Hotfix: SAM_5.0_035

Summary:
Retention procedure not working correctly
Behavior Before Fix:
The retention procedure based its decisions on the content of the file, read from metadata.ï¿½ This can cause issues and activity log can be accidentally removed.
Behavior After Fix:
The retention procedure now uses the timestamp extracted from the file name to decide if the file needs to be removed
