Event ID 5804 in Health Log

Symptom

Netwrix Auditor prompts the following error in the Health Log:

Event ID: 5804
Unable to establish an activity session.

Causes

Review the Reason field in the event description.

TIP: Here are the examples of the possible Reason values:

  • Reason: 0x80190190 Bad request (400)

    • Unable to find requested credential by username

    • Unable to find requested resource by DnsHostName

    • Unable to find requested activity: %activity%

  • Reason: 0x80190193 Forbidden (403)

    • Either NA or NPS has invalid certificate.

  • Reason: 0x80072EE2 The operation timed out

Possible Resolutions

Depending on the environment, review the following options:

  • Review the NPSAM logs to learn more about possible issues.

  • If needed, increase the timeout value in Auditor:

    1. Locate the CredentialResolverService.acconf file in the following folder:

      C:\Program Files (x86)\Netwrix Auditor\Audit Core\

    2. Locate the MaxWaitForActiveStatusS node and alter the timeout value (seconds):

      <a n="MaxWaitForActiveStatusS" v="300"/>

      Save the changes and restart the NwCfgServerSvc service to apply the changes.

    IMPORTANT: Auditor overwrites .acconf files on NA upgrades and repairs.
