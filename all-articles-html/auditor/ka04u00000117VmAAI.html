Failed Logon Attempts after Recent Service Account Password Change

Symptom

A Netwrix Auditor service account is generating failed logon events after the recent password change.

Cause

The service account password was not updated for separate collectors (Password Expiration Notifier, Event Log Manager and Inactive User Tracker) with monitoring plans set up.

Resolution

A service account password is not propagatedï¿½automatically in Password Expiration Notifier, Event Log Manager and Inactive User Tracker collectors.

For Event Log Manager, Inactive User Tracker and Password Expiration Notifier:

  1. Open the corresponding application, select your monitoring plan and click Edit.

  2. In the General tab, specify new credentials for your service account.
