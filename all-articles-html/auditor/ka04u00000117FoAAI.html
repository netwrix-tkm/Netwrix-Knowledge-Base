Symptom

Cause

It is a known issue fixed in 10.5, build 10977.

Resolution

  1. Stop Netwrix Auditor for File Servers Audit Service.

  2. Find the FileStorageAuditor folder in your Working Folder and rename it to FileStorageAuditorOld.

    TIP: To find your Working Folder, go to the Health Status section of your Netwrix Auditor, click Open diagnostic logs folder under the Working Folder frame and go one level up.

  3. Delete the memory dump files from C:\Windows\Temp\ generated by crashes.

  4. Upgrade to new build.

    IMPORTANT: If you're already using v10.5 build 10977, please skip this step.

  5. Restart Netwrix Auditor for File Servers Audit Service.

  6. Run a Data Collection by selecting an update on the monitoring plan.
