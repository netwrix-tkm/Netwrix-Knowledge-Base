Best Practices for Securing Netwrix Auditor

Question

What are the best practices for securing Netwrix Auditor?

Instructions

  • Limiting access to the Netwrix Auditor host.

  • Maintaining roles in Netwrix Auditor.

  • Monitoring Netwrix Auditor services.

  • Enabling native Microsoft security tools.

  • Auditing related systems via Netwrix Auditor.

  • Doing regular offline backups of Long-Term Archive.

Limiting access to the Netwrix Auditor host

Consider using Restricted Groups when applying group membership and User Rights Assignment policy settings to allow access to the Netwrix Auditor host to a limited group of users. For additional information, refer to the following articles: Description of Group Policy Restricted Groups and User Rights Assignment.

Maintaining roles in Netwrix Auditor

Netwrix Auditor provides a flexible role-based access (RBAC) model, which restricts what each user can do in Netwrix Auditor according to their role. For additional information about RBAC, refer to the following article:� Role-Based Access and Delegation.

Monitoring Netwrix Auditor services

Ensure that the critical Netwrix Auditor services are always up and running: the Netwrix Auditor Configuration Service and the Netwrix Auditor Archive Service. To do this, you can use Netwrix Service Monitor, a freeware tool for monitoring critical Windows services. The Service Monitor tool enables monitoring all startup services on multiple servers and can send e-mail alerts if one or more services stop unexpectedly. For additional information, refer to the following page: Netwrix Service Monitor.

Enabling native Microsoft security tools

  • To secure your SQL database data, enable transparent data encryption. For additional information, refer to the following article:� Transparent data encryption (TDE).

  • To secure your Long-Term Archive, enable BitLocker encryption. For additional information, refer to the following link:�: BitLocker Overview.

Auditing related systems via Netwrix Auditor

  • Enable configuration and logon auditing for your SQL Server:

    • Enable alerts for logon activity, roles, and db_owner role changes for SQL databases.

  • You can enable changes to Local Users and Groups, services, and software-installation auditing for your SQL Server and Netwrix Auditor servers:

    • Configure alerts for changes to log clearance and Local Administrator groups.

    • Enable session recording for SQL Server and Netwrix Auditor host via User Activity monitoring.

    • Configure alerts for SQL Management Studio or Netwrix Auditor launch.

    • Configure alerts for logons to the SQL server and Netwrix Auditor host.

  • Enable auditing of Long-Term Archive:

    • Exclude the Netwrix data-processing account from the monitoring scope.

    • Configure alerts for all read/modify/delete events as well as for failed activities.

Making regular offline backups of the Long-Term Archive

To avoid data loss in case of unexpected archive corruption, malicious actions, ransomware, and other circumstances, make sure to regularly back up your Long-Term Archive. Options include off-site or cloud backups to ensure the intergrity of LTA data.
