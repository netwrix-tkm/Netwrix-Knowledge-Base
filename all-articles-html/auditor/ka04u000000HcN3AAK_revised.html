Exporting Information on Account Lockout Events

Overview

Netwrix Account Lockout Examiner helps IT administrators discover why an Active Directory account keeps locking out. By identifying the lockout reason, administrators can quickly restore normal operations and prevent future occurrences.

Instructions

Step 1: Set Up Auditing

To monitor account lockout events effectively, enable the appropriate audit policies on the target machines:

  • Advanced Audit Policies:
    • Audit Credential Validation (Event ID: 4776, Failure)
    • Audit Kerberos Authentication (Event ID: 4771, Failure)
    • Audit Other Account Logon Events (Event ID: 4776, Failure)
    • Audit User Account Management (Event ID: 4740, Success)
    • Audit Logon (Event ID: 4625, Failure)
    • Audit Account Lockout (Event ID: 4625, Failure)
  • Basic Audit Policies:
    • Audit Logon Events (Event ID: 4625, Failure)
    • Audit Account Logon Events (Event IDs: 4776, 4771, Failure)
    • Audit Account Management (Event ID: 4740, Success)

Step 2: Download and Install Netwrix Account Lockout Examiner

Download the application from the Netwrix website onto a computer within the domain where lockouts occur. Run the application and accept the end-user license agreement when prompted.

Step 3: Examine Account Lockouts

Once installed, use Netwrix Account Lockout Examiner to analyze lockout events. Select the option to examine all domain controllers to detect potential lockout reasons. After the examination completes, the tool will present a list of reasons for the account lockout.

Step 4: Export Data

To export filtered data on account lockout events, click Export Data in the application. You can choose to export the data in PDF or CSV format for further analysis.

Additional Notes

IMPORTANT: Ensure that auditing is properly configured on all domain controllers to capture relevant account lockout events.
