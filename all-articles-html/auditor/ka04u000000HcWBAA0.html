Errors: "Database '...' cannot be opened because it is offline", "Database '...' is already open and can only have one user at a time".

You get the following error(s) in summary emails and in the Netwrix Auditor System Health log: "Database 'DatabaseName' cannot be opened because it is offline" or "Database 'DatabaseName' is already open and can only have one user at a time". ï¿½ If you want to stop receiving such errors, you need to exclude the offline / single-user access database(s) from monitoring. Please add them to omitstorelist.txt in %Netwrix Auditor install folder%SQL Server Auditing (by default, C:Program Files (x86)Netwrix AuditorSQL Server Auditing) using the following syntax: ServerInstanceName:DatabasesDatabaseName For example: SQLSRVSQLEXPRESS:Databases mpdb IMPORTANT! Please note, that if you bring the database back online / change its access mode to multi-user later, it will stay excluded from monitoring until you comment out or remove the corresponding line from omitstorelist.txt.
