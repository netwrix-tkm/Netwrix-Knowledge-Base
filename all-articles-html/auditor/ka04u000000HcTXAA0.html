Error 500.24 Internal Server Error when you try to go to web portals

The Error 500.24 Internal Server Error is received when going to the web portals (Admin Portal, Help Desk Portal, Self-service Portal).
It occurs because ASP.NET Integrated mode is unable to impersonate the request identity in the BeginRequest and AuthenticateRequest pipeline stages.
To resolve the issue, Change the App pool for the site(Default Web Site) to Classic instead of integrated mode:
    1. Open the IIS manager
    2. In the left pane, expand the Sites node>li>
    3. Select the Default Web Site node
    4. In the right pane, click "Basic Settings"
    5. Click "Select", from the drop down choose "Classic .Net AppPool"
    6. Click "OK"
