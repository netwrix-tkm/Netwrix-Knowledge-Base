Transport Failed to Connect to Server

Symptom

When verifying Notifications settings in Netwrix Password Expiration Notifier, the following error is prompted:

Test failed: The transport failed to connect to the server.
Exception has been thrown by the target of an invocation.

Causes

  • TLS 1.2 is misconfigured in the Netwrix server.

  • The implicit SSL connection mode is enabled.

  • SMTP settings are misconfigured.

  • The monitoring plan is corrupted.

Resolution

  2. Uncheck the Use the Implicit SSL connection mode checkbox in the Notifications tab in PEN, and verify the SMTP settings.

  4. Review the list of ports and protocols required for PEN to operate ? refer to the following article for additional information: Tools ? Password Expiration Notifier ? v10.6.

  5. Recreate your PEN monitoring plan ? it is required to perform the setup manually as there's no option to copy the existing plan automatically.
