Incorrect Date and Time Format in Netwrix Auditor for Network Devices Logs

Symptom

Activity Records and logs collected by Netwrix Auditor for Network Devices contain incorrect date and time values.

Cause

The time zone of network devices might be different from the Auditor Server time zone.

Resolution

  • Customize the time format in the events at the device end so it does not include UTC.

    Oct 27 10:51:05 instead of Oct 27 10:51:05 UTC.

    For Cisco devices, use the following instructions: service timestamps ? Cisco 🡥

  • Adjust the time zone settings in the Netwrix Auditor configuration file.

    1. On the Auditor Server, stop the Netwrix Auditor Configuration Server Service.

    2. Navigate to the Configuration.xml file and make a backup. (%Working Folder%\AuditCore\ConfigServer\Configuration.xml)

    3. Open the Configuration.xml file in an editor and search for the name/IP address of your network device as it appears in your monitoring plan.

    4. Locate the entry where audit_item_value is equal to your deviceï¿½s identifier (name or IP).

    5. Adjust the time zone by modifying the v=" " parameter next to the tz attribute.

    6. If you have multiple devices in the Network Devices plan, repeat these steps for each device.

    7. Once complete, save the file and restart the Netwrix Auditor Configuration Server Service.
