Error: Failed to check for event log existence

The following error is displayed in the session results/ daily change summary email: "Failed to check for event log existence".
This problem occurs because the loopback check functionality is active on the problematic server.
To solve the problem, perform the following test:
  1. Launch Event Viewer on the machine where Netwrix product is installed.
  2. From the Action menu, select the Connect to another computer option and specify the name of the problematic computer.
  3. Select the Connect as another user option, click the Set user button and specify the account which is used to run the Event Log Manager scheduled task (the default data processing account).
If you can connect successfully, the issue can be resolved by turning off the Network Traffic Compression option in the Event Log Manager settings, or by performing the procedure described in the following Microsoft Knowledge Base article: http://support.microsoft.com/kb/926642.
