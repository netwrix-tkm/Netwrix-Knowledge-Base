Error During Report Processing � rsErrorImpersonatingUser Running Netwrix Auditor Reports

Symptom

The following error occurred when trying to run Netwrix Auditor reports:

An error has occurred during report processing. (rsProcessingAborted)
Cannot impersonate user for data source 'DS'. (rsErrorImpersonatingUser)
Log on failed. Ensure the user name and password are correct. (rsLogonFailed)
The user name or password is incorrect.

Cause

SQL Server Reporting Services (SSRS) connection issues or insufficient permissions for the data collection account.

Resolution

  1. Make sure you are using a supported SQL Server edition. Refer to the following article: Requirements for SQL Server to Store Audit Data ? v10.6.

  2. Check the permissions for your Audit Database account. Refer to the following article: Requirements for SQL Server to Store Audit Data: Configure Audit Database Account ? v10.6.

  3. Check the permissions for the account used to collect data in your environment. Refer to the following article: Monitoring Plans � Data Collecting Account ? v10.6.

    IMPORTANT: The account should be a member of local Administrators group.

    If you use a gMSA account for data collection, refer to the following article for additional information: Requirements � Use Group Managed Service Account (gMSA) ? v10.6.

  4. Check your Report Services configuration. Refer to the following article: Deploying the Report Server Database.

  5. Check the permissions for your SSRS Account. Refer to the following article: SQL Server Reporting Services: Configure SSRS Account ? v10.6.

    IMPORTANT: The account should be a member of local Administrators group.

  6. On the computer that hosts the Auditor Server, open the Services snap-in and restart the Netwrix Auditor Management Service.

  7. Make sure you provided valid credentials to connect to your Audit Database in Netwrix Auditor. For that, open Netwrix Auditor, navigate to Settings > Audit Database, click Configure under Audit Database settings and verify your credentials.

NOTE: Netwrix also recommends upgrading the product for the latest version to avoid SSRS-related issues.
