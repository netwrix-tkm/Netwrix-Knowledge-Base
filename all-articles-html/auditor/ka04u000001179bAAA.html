Information About Foreign Group Members (Foreign Security Principals) Missing in Active Directory State-in-time Reports

Symptom

The Active Directory state-in-time reports based on historical snapshots do not contain data for foreign group members (Foreign security principals).

Cause

An Active Directory domain contains the information about the objects and their attributes belonging to this domain. The Group Members report uses a cross-domain membership feature, it means that data collected in different monitoring plans then aggregated to show the final membership. Thus, the product cannot work with historical snapshots, because there is no guarantee that historical snapshots for other domains are also uploaded and all of them were taken at the same time, i.e., the data from other monitoring plans that could be collected at different time cannot be shown in the reports.

Resolution
