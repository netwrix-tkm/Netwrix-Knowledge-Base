Error: Audit Trails Are Incorrect

! This article is currently under review. Some of the information below may not be up to date.

Symptom

The following error occurs in the File Servers monitoring plan in Netwrix Auditor:

 
            Audit trails are incorrect.

Cause

Improper audit settings configuration in the audited environment.

Resolution

Follow the steps below to resolve the issue:

  1. On your file server, navigate to a problematic folder/share; right-click it and select Properties.

  2. Select Security tab -> Advanced.

  3. On the Audit tab, disable inheritance.

  4. Then, confirm that Security principals are configured according to Netwrix documentation: Data Source Configuration � File Servers � Configure Object-Level Access Auditing � v10.6 (Successful Changes and Failed Reads).

  5. Click Apply. The server will propagate all permissions to the target folders.

Setting up permissions manually will ensure Netwrix is detecting the settings and correct Security log events are written on the File Server.
