The event log file is corrupted. (Error number: 0x80004005)

In summary reports you see the following error: Event log read failure. Error details: The event log file is corrupted. (Error number: 0x80004005)
The event log has been corrupted. If you are using the product without agents, the product uses the Application log and if you are using the product with agents, the product uses its own log located under Event viewerApplications and Service Logs Netwrix Non-owner Mailbox Access Agent.
Please perform steps from the following Microsoft Article. If it does not helped, please also perform the following steps:
  • Disable the Windows event log service on the target Exchange server
  • Reboot the server
  • Stop the Netwrix Non-owner Mailbox Access Reporter Agent service
  • Renamed the C:WindowsSystem32winevtLogsNetWrix Non-owner Mailbox Access Agent to NetWrix Non-owner Mailbox Access Agent.old
  • Enable the Windows event log service, Netwrix Non-owner Mailbox Access Reporter Agent service, reboot the server.
