How to fix problems related to installing/uninstalling Compression Service on target host

Symptoms

  1. When removing Item, Data Source or Monitoring Plan from Netwrix Auditor, it's Compression Service Agents are not removed automatically from the Servers that are no longer being monitored.
  2. After the upgrade of Netwrix Auditor, there are errors in Health Log that indicate that Compression Service Agents cannot be updated automatically.

Causes

Installation - Possible causes for an issue during installation are: there is a previous version of the Compression Service that cannot be uninstalled correctly, OS settings prevent installation of .msi package (e.g. Domain Policies, no permissions for Netwrix Account)
Uninstllation - Possible causes for an issue during removal are: Missing/corrupt installation package, corrupt registry keys that point to a nonexistent folder.

Solution

In this case, the Agents must be removed manually. Netwrix Auditor uses standard .msi packages to deploy its Compression Service Agents. In order to remove them, we recommend using Troubleshooting Software provided by Microsoft here: https://support.microsoft.com/en-us/topic/fix-problems-that-block-programs-from-being-installed-or-removed-cca7d1b6-65a9-3d98-426b-e9f927e1eb4d

NOTE: All operations must be done on the Target server, not on the server where Netwrix Auditor is installed. If there are multiple affected Target servers, the operation must be done on each one of them separately.

There are two possible scenarios:

The agent cannot be installed on the target host

  1. Run the troubleshooter program that was downloaded form Microsoft Website
  2. click Next and select Uninstalling option,
  3. find and select the corresponding name of the Compression Service Agent in the list and follow the steps to complete the procedure.
  4. If data collection with compression service is enabled for this server, Compression Service will then be automatically installed by Auditor.
If the agent name is not in the list, then run the troubleshooter again and select Installing option, select the name of the agent in the list and complete the procedure.


The agent cannot be uninstalled on the target host

  1. Run the troubleshooter program that was downloaded form Microsoft Website
  2. click Next and select Uninstalling option,
  3. find and select the corresponding name of the Compression Service Agent in the list and complete the procedure.
If you follow the instructions, but the problem still persists, contact Netwrix Technical Support.
