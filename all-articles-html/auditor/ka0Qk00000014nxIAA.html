Why Does Netwrix Service Account Access KDS Root Key Object?

! This article is currently under review. Some of the information below may not be up to date.

Question

Why does Netwrix Service Account access the KDS root key object on a domain controller considering that this object is used to store the secret on which the password calculations for service accounts are based.

Answer

This behavior is tied to State-in-Time data collection. The KDS root key is one of the objects collected with State-in-Time snapshots. In this case, the product accesses only audit data associated with the object and does not collect any key values.
