Netwrix Auditor Health Log Contains EventID 3230

Symptom

Netwrix Auditor (Auditor) System Health Log contains EventID 3230 for a SharePoint Online monitoring plan.

Cause

The lock status for a site collection indicates that this is a personal site collection that has been locked. Personal site collections are not intended to be collected, and the error with No access lock status can be fixed by unlocking the site. Auditor for SharePoint is intended as an enterprise solution and should be configured for public documents only.

IMPORTANT: This event does not affect data collection.

Resolution

As a workaround, you can exclude personal site collections from being audited by editing the OmitSitScStoreList.txt file located in the following default path:

C:\ProgramData\Netwrix Auditor\

NOTE: To determine your Working Folder location, you can run the following PowerShell command. If this script does not return a value, then your Working Folder is the default location of C:\ProgramData\Netwrix Auditor\.

For additional information on how to configure your SharePoint Online monitoring scope, refer to the following article: SharePoint Online Monitoring Plans – Monitoring Scope – v10.6.

IMPORTANT: The folder associated with Netwrix Auditor must be excluded from antivirus scanning. See the Antivirus Exclusions for Netwrix Auditor knowledge base article for additional information.
