Changing the name of your SQL Server causes database related errors

When you change the name of your SQL Server, it can lead to various database-related errors. This article outlines the necessary steps to ensure a smooth transition and avoid potential issues.

Default Database Name

The default name for the original database was NetwrixAIC . However, it could have been Customized .

SQL Server Connection Details

When configuring the SQL Server connection, ensure you provide the correct database information:

  1. Server: Enter the database server hostname (NetBIOS name, FQDN, or IP address) with the instance name or non-standard port, if applicable, in one of the following formats:
    • No named instance: [SQLHostName], for example NT-SQL02
    • Named instance: [SQLHostName]\[SQLInstanceName], for example NTSQL02\Netwrix
    • No named instance with non-standard port: [SQLHostName],[PortNumber], for example NT-SQL02,72
    • Named instance with non-standard port: [SQLHostName]\[SQLInstanceName],[PortNumber], for example NT-SQL02\Netwrix,72
  2. Database: Enter the name of the database. By default, the database responsible for the Netwrix Auditor look and feel is Netwrix_CommonDB . If you renamed this database, provide the new name.

Conclusion

Changing the name of your SQL Server requires careful attention to the database connection details to avoid errors. Ensure that you follow the guidelines provided above to maintain a seamless operation.
