How To: Change the portal URL in workflow email notifications

Applies To

Directory Manager 11

Business Case:

I have configured the Directory Manager Self-Service portal to use a different hostname than the default one, which was created using the server�s FQDN. However, the email notifications that Directory Manager generates for workflows still contain the old FQDN-based portal URL.

Is there a way to change this URL in email notifications?

Solution:

Directory Manager enables you to link the URL of a Self-Service portal with a workflow email. Email notifications for the workflows contain this URL to redirect recipients to that portal for taking any necessary action, such as approving or denying workflow requests.

In case the portal URL is changed, you must link the new URL to your workflows.

Methodology:

To change the Self-Service portal URL for a workflow, follow these steps.

  1. In the Directory Manager Admin portal, click the Identity Stores tab.
  2. On the Identity Stores tab, click on the three dots button to edit the properties.

  1. Click the Workflows tab. Select the workflow that you want to change the portal URL for, and click Edit.
    For example, to change the URL in email notifications that alert group owners to approve or deny membership requests, select �Workflow to Join a Group�.

  2. Click on the Portal URL box and select the portal URL you want to use.



  1. Click Update Workflow and Save your changes. Now the URL that will be embedded in the email will be the one you have specified here. You can apply this change to all your workflows one by one.
