Compliance Report Error: Not Run

Symptoms:
�
When the customer is running the report Netwrix CIS Red Hat Enterprise Linux 7 Server Benchmark - Level 1 and/or Level 2 the results displays "not run".
�

��

��
Cause: This particular report's rule # 1.1.21 or "Ensure sticky bit is set on all world-writable directories" is producing a ton of output, "overloading" the report and causes it to "not run". This was identified by splitting the report into segments until we found which rule was causing the problem.
��

��
Resolution: Limiting the output to 100 lines.
��

�
�df --local -P | awk {'if (NR!=1) print $6'} | xargs -I '{}' find '{}' -xdev -type d \( -perm -0002 -a ! -perm -1000 \) 2>/dev/null | head -n 100
�
Note: I believe this could also apply to other rules that produces too much output, not necessarily for this rule only.
