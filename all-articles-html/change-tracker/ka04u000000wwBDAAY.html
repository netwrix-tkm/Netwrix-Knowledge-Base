MongoDB Service Terminated Unexpectedly

Symptom

The MongoDB Service is not stay running and stops after each startup.

Cause

The issue occurs if a database was corrupted and this can happen when a server is shutdown unexpectedly as MongoDB doesn't have time to shutdown safely.

Resolution

  1. On the computer that hosts Change Tracker software, Stop the MongoDB service.

  2. Open a Command Prompt, ensure that you run it as an Administrator.

  3. Enter the following command:cd "C:\Program Files\NNT Change Tracker Suite (Netcore)\Gen7\MongoDB\bin".

  4. Enter the following command:mongod.exe --dbpath "C:\ProgramData\Change Tracker Generation 7 (Netcore)\MongoDB\db" --repair.

  5. Enter the following command:iisreset.

  6. Close the command prompt window.
