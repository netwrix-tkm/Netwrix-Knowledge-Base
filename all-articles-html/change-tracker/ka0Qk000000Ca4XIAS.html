How to Upgrade your Mongo Database (Windows)

IMPORTANT: This article is outdated and is no longer applicable.

This is a guide on upgrading your Mongo Database to a newer version, See the following link for version downloads of MongoDB:

https://www.mongodb.com/download-center?jmp=nav#previous

NOTE: In order to upgrade your MongoDB, you will need to upgrade to the latest instance of your current Mongo Version. For example, in order to upgrade from v3.2.10 to v3.4.17, you will need to fully upgrade to v3.2.21 prior to going to v3.4.

Step-by-step guide

  1. Prior to Upgrading, it may be best to run a full backup of your Mongo Database in case you have issues with the upgrade.
  2. Stop the MongoDB, Agent, and IIS Service.
  3. Backup the \conf folder from your MongoDB Install Directory (Default LocationC:\Program Files\NNT Change Tracker Suite\Gen7\MongoDB).
  4. Uninstall older MongoDB Version.
  5. Run the new MongoDB Installer.
  6. In the Installer, Select "Custom Installation".
  7. Click "Browse..." and select your MongoDB Install Directory(Default LocationC:\Program Files\NNT Change Tracker Suite\Gen7\MongoDB).
  8. Proceed and Install (If it asks to install compass, uncheck and continue).
  9. Start the MongoDB, Agent, and IIS Service.
  10. Verify in Hub Service log that it is currently running latest Mongo Version.(Location C:\inetpub\wwwtooy\Change Tracker Generation 7 Hub\log) See below for Log message:
