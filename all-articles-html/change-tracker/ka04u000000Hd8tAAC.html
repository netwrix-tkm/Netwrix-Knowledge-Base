How to Update Change Tracker Agents from the Hub Server

Change Tracker has an Auto Update process for windows based agents. This process is initiated from the Hub server.
�

Step-by-step guide

The following steps below will show you how to upload an update to your Change Tracker Server for Agents to be updated.

�

1 - On our website, you need to download the agent zip file and agent updater zip file. Move both the Agent and Agent updater zip files to the same directory

�

2. Once you have downloaded the two files, and moved them to the same directory - Log into your change tracker console, and go to the "Settings" tile.

�

3. On the Left - Look for the "Agent Updates" Navigation button - Click on that - On the agent updates screen, click on "Upload an Update" - Select BOTH the Updater file and the matching agent version, and click open

�

4 - The agent files will upload - Click on the "Upload Files" button and they will appear in the list.

�

�

After this, the agents are ready to be assigned to the proper group to receive the updates. Follow the steps below to assign them to the proper group

�

�

Step-by-step guide

The following steps below will show you how to assign an uploaded update package the the correct group of agents to be updated.

�

1 - On "Settings" tile - Select the "Groups" button. Select the group that the agent update needs to be assigned to (it will be highlighted in blue when selected) and click on the "Agent Software Updates" tab. From here, you can set the update schedule by pressing the "Define the update schedule" button.

�

2 - Define an update schedule that is of your liking, and select the agent package from the drop down. After Clicking update, your schedule will take effect immediatly.

�

The next time the agents phone into the console within the update schedule you have defined, they will receive the update package and attempt to upgrade to that version.
