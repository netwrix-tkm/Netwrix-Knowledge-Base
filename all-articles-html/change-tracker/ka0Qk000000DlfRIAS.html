Rolling-Log Fix: "Hub Offline, connection retries exhausted (The remote server returned an error: (403) Forbidden.)"

This problem occurs when the Netwrix Gen 7 Agent service is not able to authenticate/communicate with the Change Tracker Server as the service user assigned does not have enough privileges.

2019-12-03 14:48:31,110 [10] INFO Message - Hub Offline, connection retries exhausted. Sleeping until 03/12/2019 13:49:53 (The remote server returned an error: (403) Forbidden.)

Instructions

  1. Remote onto the agent device raising this error.

  2. Stop the Netwrix Gen 7 Agent service and click into the properties of the service.

  3. Click on the Log On tab.

  4. Select This account and assign a user with more privileges such as administrator for example and press OK.

  5. Navigate to your Netwrix Agent Directory Which contains the Rolling-Log Files - Which is defined in this article -Netwrix Software Problem? What Info Should I Provide?

  6. Right-click and edit the Hubdetails.xml file.

  7. Remove the Tags as they are the agent password which has been encrypted

  8. In the place of the Tags, enter the Tags - See example hubdetails below. The default password is used as an example.

  9. In between the password entries, enter your Agent Account password as defined on your Netwrix Hub Server

  10. Save the file out, replacing the old one.

  11. Start the Netwrix Gen 7 Agent service.
