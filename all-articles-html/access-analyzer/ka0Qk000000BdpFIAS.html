Cause

The Proxy and File Action Module are using the same port.

Resolution�

The ports for the FSAA scan have to be adjusted. Set [port of the Applet] +1 = [port for the cert exchange].�

To change the Applet port on the proxy,

1) Stop the "Netwrix Access Analyzer FSAA Proxy scanner" Service�

2) To update Run this command to change the port based on the updated Applet port. (also will depend on the path the service is installed.)
Example:�sc config StealthAUDITFSAA binPath="\"C:\Program Files (x86)\STEALTHbits\StealthAUDIT\FSAA\FSAAAppletServer.EXE\" -e 8769"�

3) Start the service in services�

4) Verify in task manager that it is running on this port In the FSAA scan configuration set the Applet port to the new port In the FSAA scan configuration set the port for the cert exchange to this port +1�

Once the scan is run, the proxy service will free up the port for the File Action Service.
