Error: Object Named PK_SA_SPAA_Exceptions Is Already in Database

IMPORTANT: This is a work-in-progress article. The product R&D team has yet to review some or all of information below.

Symptom

Netwrix Enterprise Auditor prompts the following error when you run the 5-SPAA_BulkImport job:

There is already an object named 'PK_SA_SPAA_Exceptions' in the database.
Could not create constraint or index.
        
Cannot find the object "dbo.SA_SPAA_Exceptions" because it does not exist or you do not have permissions.
        
Exception Error:
Invalid object name 'dbo.SA_SPAA_Exceptions'.
SA_SPAA_SiteCollectionsView: NOT CHANGING
SA_SPAA_WebAppPoliciesView: NOT CHANGING
SA_SPAA_SiteCollectionsTraversalView: NOT CHANGING
SA_SPAA_EffectiveAccessView: NOT CHANGING
SA_SPAA_PermissionsView: NOT CHANGING
SA_SPAA_DirectPermissionsView: NOT CHANGING
SA_SPAA_LocalGroupMembersView: NOT CHANGING

Causes

Refer to the possible causes in your environment:

  • Multiple copies of the SPAA_Exceptions job exist and run simultaneously.

  • Instances of SPAA_Exceptions and SPAA_BulkImport are running simultaneously.

Resolution

Enterprise Auditor requires one instance of the SPAA_Exceptions job to identify all potential high risk conditions. Review and perform the following steps to resolve the issue:

  1. Ensure that only one copy of the SPAA_Exceptions job is enabled in your Enterprise Auditor environment.

  2. Ensure that the SPAA_Exceptions job does not overlap with any scheduled SPAA_BulkImport job.
