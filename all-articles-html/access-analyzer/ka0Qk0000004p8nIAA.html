Error: Could Not Open Registry Key for Reading

IMPORTANT: This is an unvalidated article. The product R&D team has yet to review some or all of information below.

Symptoms

The following symptoms are present in your Netwrix Enterprise Auditor environment:

  • A File System Activity Scan (FSAC) job in your Netwrix Enterprise Auditor prompts the following error:

    Error during processing:  
Could not open registry key SYSTEM\CurrentControlSet\Services\SBTLogging\Parameters for reading: 
The system cannot find the file specified (2); local: False

  • Enterprise Auditor does not generate new reports or fails to add new data to reports.

Cause

The Netwrix Activity Monitor agents are not installed in the affected hosts.

Resolution

Refer to the following steps to resolve the issue:

  1. Manually install the Activity Monitor agents on the affected hosts. Refer to the following article for additional information: Manually Install the Activity Agent � v7.1.

  2. Configure Activity Monitor to monitor respective hosts.

    IMPORTANT: The used server names for the monitored servers should match the server names in the Host Management list. Refer to the following steps to edit the host name:

    • In the Host Management menu of Enterprise Auditor, select the host and click View/Edit Host in the right pane. Click Edit to introduce changes to the details.

    • Alternatively, you can use the Report Hostname as feature in AIC. Refer to the following article for additional information: Output Types � Additional Properties Tab � v7.1.

  3. Run the 1-FSAC system scans. Verify that all hosts can be successfully scanned without the missing registry key error.

  4. Review the relevant reports.
