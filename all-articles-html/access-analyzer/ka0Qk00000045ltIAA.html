When attempting to schedule a task, "Unable to access Scheduler: Could not load task scheduler" is shown.

Summary: How to resolve the scheduled task error "Unable to access Scheduler: Could not load task scheduler"

Issue: When creating a scheduled task in StealthAUDIT or Windows Task scheduler, a pop-up "Unable to access Scheduler: Could not load task scheduler" appears

Instructions: How to configure Task Scheduler permissions

1: Check if UAC is turned on, if it is, disable it
2: The account configured under Settings > Schedule in StealthAUDIT should be a local administrator, or have the appropriate minimum permissions to schedule a task

  • Minimum permissions would require that the account be granted "Logon as a batch job" under the StealthAUDIT server's Local Security Policy.
    1. Perform a Windows search for secpol, or right-click Start > Run secpol.msc > OK, and open the Local Security Policy
    2. Expand Local Policies > User Rights Assignment
    3. Scroll down to the "Logon as a batch job" policy and double-click to edit the properties
    4. On the Properties window, click the Add User or Group button and add the desired user or group accounts
    5. Click Apply
