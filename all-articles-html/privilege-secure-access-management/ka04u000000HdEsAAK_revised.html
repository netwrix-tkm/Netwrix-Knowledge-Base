Disable MFA for Users or Groups

Overview

This article provides guidance on how to disable Multi-factor Authentication (MFA) for individual users or groups of users in SbPAM. While disabling MFA may be necessary in certain scenarios, it is important to understand the security implications and follow best practices to mitigate risks.

IMPORTANT: Disabling MFA is strongly discouraged as it significantly reduces the security of user accounts. MFA is a critical security measure that protects against unauthorized access, even if passwords are compromised. Consider alternative solutions before proceeding.

Instructions

  1. Log in to SbPAM as an Administrator.
  2. Navigate to Users & Groups in the sidebar navigation.
  3. Click on a user or a group, then navigate to that principal's Authentication Connector tab.
  4. Click Not Required to disable the MFA requirement for the selected user or all users in the group.

Best Practices

  • Before disabling MFA, evaluate whether alternative solutions, such as conditional access policies, can address your requirements without compromising security.
  • Ensure that users impacted by the change are informed and provided with guidance on maintaining account security.
  • Regularly audit user accounts and access permissions to identify potential vulnerabilities.
  • Re-enable MFA as soon as possible to restore the security of affected accounts.
