Hotfix: SbPAM 3.7 Hotfix 1 (3.7.559)

Fix issue with local managed Windows accounts being unable to login after creating a session. With the Create checkbox checked for the activity the password was not getting set in SbPAM so the proxy was using an old password when logging in.
