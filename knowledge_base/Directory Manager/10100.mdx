## Applies To

Directory Manager 11

## Overview

When you integrate Netwrix Directory Manager (formerly GroupID) with an existing Active Directory environment, some groups may have been created directly in Active Directory. These groups might not have the __displayName__ attribute populated, since it is not required by Active Directory.

Directory Manager, however, requires the __displayName__ attribute because it uses this value to populate other fields, such as Alias and CN. If you try to save changes to one of these groups in the Self-Service Portal (SSP) and __displayName__ is missing, you will receive an error. You cannot save changes until __displayName__ is populated or the requirement is removed.

This article describes a workaround that allows you to save changes to these groups by making the __displayName__ attribute optional in the portal design.

![Error message displayed in Self-Service Portal when displayName is missing](https://nwxcorp.file.force.com/servlet/rtaImage?eid=ka0Qk000000DvbZ&feoid=00N0g000004CA0p&refid=0EMQk00000BSYlB) ![Error details dialog showing missing displayName attribute](https://nwxcorp.file.force.com/servlet/rtaImage?eid=ka0Qk000000DvbZ&feoid=00N0g000004CA0p&refid=0EMQk00000BSWjO)

## Instructions

* In the Directory Manager Admin Center, go to the __Applications__ tab. Open the settings for the application where you want to remove the __displayName__ requirement by clicking the __Settings__ button (top right corner of the application card).

![Open application settings in Directory Manager Admin Center](https://nwxcorp.file.force.com/servlet/rtaImage?eid=ka0Qk000000DvbZ&feoid=00N0g000004CA0p&refid=0EMQk00000BSYmn)

Under Design Settings, click the domain name.

![Select domain under Design Settings](https://nwxcorp.file.force.com/servlet/rtaImage?eid=ka0Qk000000DvbZ&feoid=00N0g000004CA0p&refid=0EMQk00000BSYoP)

Go to the Properties tab and set the Directory Object to Groups.

![Set Directory Object to Groups](https://nwxcorp.file.force.com/servlet/rtaImage?eid=ka0Qk000000DvbZ&feoid=00N0g000004CA0p&refid=0EMQk00000BSYq1)

Edit the General field by clicking the pencil icon.

![Edit General field](https://nwxcorp.file.force.com/servlet/rtaImage?eid=ka0Qk000000DvbZ&feoid=00N0g000004CA0p&refid=0EMQk00000BSYrd)

Edit the DisplayName field by clicking the pencil icon in the Design category window.

![Edit DisplayName field](https://nwxcorp.file.force.com/servlet/rtaImage?eid=ka0Qk000000DvbZ&feoid=00N0g000004CA0p&refid=0EMQk00000BSYtF)

Expand Advanced Options and uncheck the Is Required box.

![Uncheck Is Required for DisplayName](https://nwxcorp.file.force.com/servlet/rtaImage?eid=ka0Qk000000DvbZ&feoid=00N0g000004CA0p&refid=0EMQk00000BSYwT)

Click all OK buttons to save your changes, then log in to your portal. You should now be able to save changes to a group even if the displayName attribute is not populated.To apply this change to Smart Groups, set the Directory Object to SmartGroup in step 3 then complete the remainder of the steps.
