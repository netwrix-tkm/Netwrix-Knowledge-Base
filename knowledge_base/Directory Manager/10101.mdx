## Overview

This article shows how to use the Reports module in Netwrix Directory Manager (formerly GroupID) 11 to identify groups that lack a primary owner, additional owners, or any owners.

## Instructions

* In the Directory Manager application portal, click the __Reports__ tab on the left side of the dashboard page.

![Directory Manager application portal with Reports tab highlighted on the dashboard](https://nwxcorp.file.force.com/servlet/rtaImage?eid=ka0Qk000000Dxof&feoid=00N0g000004CA0p&refid=0EMQk00000BSZAz)

When the Reports module opens, click the Group Reports tab on the left side of the page.

![Group Reports tab selected in the Reports module](https://nwxcorp.file.force.com/servlet/rtaImage?eid=ka0Qk000000Dxof&feoid=00N0g000004CA0p&refid=0EMQk00000BSZ69)

To find groups without a primary owner, run the report titled Groups with no owner.

![Groups with no owner report selected in Group Reports](https://nwxcorp.file.force.com/servlet/rtaImage?eid=ka0Qk000000Dxof&feoid=00N0g000004CA0p&refid=0EMQk00000BSZ9N)

To find groups without additional owners, run the report titled Groups without additional owners.

![Groups without additional owners report selected in Group Reports](https://nwxcorp.file.force.com/servlet/rtaImage?eid=ka0Qk000000Dxof&feoid=00N0g000004CA0p&refid=0EMQk00000BSZ7l)

To find groups without both primary and additional owners, run either of the above reports. In the Report Generation wizard, replace the LDAP query with your custom query as needed.

![Report Generation wizard with LDAP query field](https://nwxcorp.file.force.com/servlet/rtaImage?eid=ka0Qk000000Dxof&feoid=00N0g000004CA0p&refid=0EMQk00000BSXh5)

Complete the wizard. The generated report will show groups that do not have a primary owner or additional owners.
